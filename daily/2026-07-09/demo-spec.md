# Demo 设计说明 — 2026-07-09

## 1. Demo 名称

**Untangle（思绪解结）** — 把一段口述乱麻整理成「可确认的思路卡」。

## 2. 一句话定位

对着它把纠结的事一口气说出来，它帮你分清哪些是**决定**、哪些**还没想清**、你在哪**改了主意**，你逐条点头确认后，才生成一张思路卡——语音不是更快的键盘，是边想边说的白板。

## 3. 目标用户

习惯「用嘴思考」的知识工作者：脑子转得比手快、常在通勤 / 走路 / 睡前迸出想法、又容易跑题反悔的创始人、PM、研究者、写作者、ADHD 群体。

## 4. 使用场景

一件纠结/需要理清的事——「要不要接这个外包项目」「给团队安排下周活」「复盘刚才和客户的通话」——脑子里一团乱、打字跟不上思路、丢给 ChatGPT 又得到一段抹平了纠结、也没让自己确认的漂亮总结。

## 5. Demo 策略分型（重要）

- 本机会属于：**☑ 可视化 / 交互类产品**（核心价值就是一条「说→拆解→确认→成卡」的可视化交互闭环）。
- 采用的演示方式：**直接做可点击交互 Demo**。其中「口述输入」这一环，因不接麦克风/STT 外部 API，采用**脚本化「模拟口述」**（预置独白逐句流入，模拟 200ms 低延迟与「按思路单元分段」的轮次检测体验）；「AI 分析」这一环采用**确定性 mock 分析**（思路单元与矛盾对为预标注数据），并在界面显式标注「示例分析结果」。
- 演示的是**我们分析出的创新切入点**——「思路单元分类 + 矛盾/改主意检测 + 逐条人工确认 → 思路卡」，**不是** Willow（口述即输入）或 AudioPen（乱麻洗成漂亮文本）的克隆。

## 6. 核心流程

（用户从进入到「看懂价值」的主路径，3 分钟内可完成）

1. **选场景 + 模拟口述**：从 3 个纠结场景中选一个，点「▶ 开始口述」，独白逐句流入转写区（模拟低延迟 + 按语调/节奏把独白切成「思路单元」气泡，呼应报告里 AssemblyAI 的轮次检测）。
2. **解结（Untangle）**：点「整理我的思绪」，每个思路单元被分类为 ✅决定 / ❓待定 / ▶行动 / 💭背景，并**高亮出「你先说 X、后改口 not-X」的矛盾/改主意对**并连线——这是与洗稿工具最不同的一屏。
3. **逐条确认（人在方向盘）**：用户对每条单元确认/改类型/删除；对每个矛盾对**拍板「我最终怎么想」**。进度条显示「已确认 N/总数」。
4. **生成思路卡（Clarity Card）**：全部确认后生成结构化卡片——【我决定了】【还没想清】【行动项】【我改了什么主意】，一键复制；并提供 **before/after** 对比（左：原始口述乱麻；右：确认后的思路卡）直观展示增量价值。

## 7. 页面结构（<= 3 个主要页面）

单页应用，3 个主要视图（步骤）通过顶部步骤条切换：

- 视图 1 — **口述台（Intake）**：场景选择 + 模拟口述回放 + 实时思路单元气泡。
- 视图 2 — **解结台（Untangle & Confirm）**：思路单元分类看板 + 矛盾对高亮 + 逐条确认交互（本 Demo 的核心屏）。
- 视图 3 — **思路卡（Clarity Card）**：结构化成卡 + before/after 对比 + 复制。

## 8. 关键交互

- 「▶ 开始口述」：逐句流入，气泡按思路单元出现，底部显示「延迟 ~200ms · 按语调分段」的模拟状态条。
- 思路单元卡片：点击可切换类型（决定/待定/行动/背景）、可删除；AI 给出的初始类型带「置信度」标记与「示例分析」水印。
- 矛盾对：两条冲突单元用红色连线 + 「⚠ 你改了主意？」标签；点击展开「你先说… / 后来说…」，让用户选择「以后者为准 / 以前者为准 / 都保留为待定」。
- 确认进度条：未确认完不允许生成思路卡（强制「人在方向盘」）。
- 思路卡：分区展示 + 复制到剪贴板；before/after 左右对比。
- 「换个场景重来」：重置状态。

## 9. 使用的 mock 数据

（说明数据来源与结构，全部为 mock，不接真实后端/外部 API）

- 3 个纠结场景，每个包含：`title`、`context`、一段**预写的口述独白**（`segments[]`，每段含 `text` + 出现顺序，刻意包含跑题、重复、反悔）。
- 每段 `segment` 预标注：`type`（decision/open/action/context）、`confidence`（0–1，用于展示 AI 不确定性）。
- `contradictions[]`：记录矛盾/改主意对（`fromId` → `toId` + 一句话说明），用于高亮连线。
- 分析结果为**确定性预标注**（非实时模型推理），界面显式标注「示例分析结果，真实版由模型实时生成」以避免夸大。

## 10. 明确不做什么

- 不做登录 / 用户系统。
- 不接麦克风 STT / 数据库 / 支付 / 外部私钥 / 真实 API / 真实语音模型。
- 不做生产级后端；所有数据 mock、状态仅存在于前端内存。
- 不追求「把口述洗成漂亮文本」（那是被差异化的对象，不是本产品目标）。

## 11. 成功体验标准

- 用户 3 分钟内能说出：「它解决什么（帮我把口述的纠结想清楚并确认过）、怎么用（说→拆解→确认→成卡）、增量在哪（不抹平我的矛盾、逼我拍板、我确认过而非盲信 AI）」。
- 用户能明确感到它**不是**又一个听写 / 洗稿工具——「矛盾/改主意高亮 + 逐条确认」这一屏必须让人眼前一亮。
- before/after 对比能让「乱麻 → 思路卡」的增量一眼可见。
