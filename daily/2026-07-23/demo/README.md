# 上手 Knack —— 从你自己的「失手」里，练出用 AI 的手感

一个 **AI 用法教练**。它不给你上课，而是回看**你自己**最近和 AI 的真实协作，
诊断你**反复犯**的用法失手，教你针对性的可迁移招式，并用「上手指数」量化你的长进。

> 本项目为 product-opportunity-lab 2026-07-23 的每日机会 Demo。灵感来自当日报告里
> 板块 B 唯一达标的 **AGINE Academy**（把「学会用 AI」做成游戏化闯关课）——但**不照抄**：
> AGINE 是**课程优先**的内容产品，上手是**行为诊断优先**的教练产品，攻击的正是报告点名
> AGINE 的两大软肋：「学完不知道自己会不会用」与「课程内容易过时」。

## 它解决什么问题

数以亿计的人每天用 ChatGPT/Claude 干活，但「结果平庸、来回返工、说不清问题出在自己哪」。
看课/文档学不会迁移；prompt 优化器只帮你改眼前这一句。**没有产品把「你自己的真实用法历史」
当作诊断对象**，去回答：你到底反复错在哪、怎么系统性变好。上手就是来补这个位。

## 核心机制（确定性引擎，非套壳）

- **失手签名 taxonomy**：7 类用户高频用法错误（目标含糊 / 没给范例 / 一股脑倒 context /
  不给约束 / 首稿即收 / 要事实不给依据 / 中途改需求），每类含人话解释、可复用招式、严重度。
- **行为诊断**：`src/logic/engine.ts` 对 mock 语料里每次协作打「质量分」，按 primary 失手聚合，
  以「影响力 = 严重度 × 结果损失」排序，找出你最该先练的 Top 失手。
- **可量化长进**：掌握一招后，引擎按「该失手伤害修复约 70%、残留 30%」重算全部指标——
  所以上手指数会升、但**不会瞬间满分**（习惯要练，诚实建模）。
- 同一套引擎同时驱动「诊断台」与「长进」两页，数字前后一致、随交互实时变化。

## 三个页面

1. **诊断台**：你的用法概览 + 上手指数 + 关键指标（一次过率 / 平均返工轮数 / 放弃率），
   以及按影响力排序的失手签名卡片。
2. **招式库**：选中失手的人话微课 + 可复用模板 + **用你自己那次会话做的 before/after 复盘**；
   「练这招 · 标记掌握」。
3. **长进**：生手 vs 上手后的 before/after 对比条（随你掌握的招式实时变化）+ 诚实的「仍需练习」提示。

## 本地运行

```bash
npm install
npm run dev      # 本地开发
npm run build    # 产出 dist/（vite.config.ts 已设 base:'./'，适配 Pages 子目录）
npm run preview  # 预览构建产物
```

## 明确边界（诚实声明）

- **纯前端静态 Demo**：所有数据为 **mock**（模拟用户「林果」的 20 次协作），
  **不读取你的真实 AI 历史**、无登录、不接后端 / 数据库 / 支付 / 外部 API / 真实 LLM。
- 真实产品需读取用户的 ChatGPT/Claude 使用历史（隐私 + 各家无统一导出口），是最大的工程/合规挑战；
  失手签名的自动识别在真实场景需 LLM 分类，可能误判——产品需保留「这是建议、可忽略」的余地。
