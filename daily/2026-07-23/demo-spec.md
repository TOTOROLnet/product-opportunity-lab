# Demo 设计说明 — 2026-07-23

## 1. Demo 名称
**上手 Knack** —— 从你自己的「失手」里，练出用 AI 的手感。

## 2. 一句话定位
一个 AI **用法教练**：回看你自己最近和 AI 的真实协作，诊断你**反复犯**的用法失手签名，
给出针对你的可迁移招式 + 用你自己那次会话做的 before/after 复盘，并用「上手指数」量化你的长进。

## 3. 目标用户
每天用 ChatGPT/Claude 干活、但「结果平庸、来回返工、说不清问题出在自己哪」的**非开发者 + 进阶个人用户**。

## 4. 使用场景
用户林果（内容/运营 PM）自学用 AI 到达瓶颈：能用起来，但常返工、结果只到 70 分。她打开上手，
看到对自己最近 20 次 AI 协作的诊断，发现自己反复「目标含糊 / 首稿即收 / 一股脑倒 context」，
逐个学会破解招式，看到「上手指数」和「一次过率」实时上升。

## 5. Demo 策略分型（重要）

- 本机会属于：☑ 可视化/交互类产品（在 mock 用法语料上的**交互式诊断+价值可视化**）
- 采用的演示方式：直接做可点击交互 Demo；因真实产品需读取用户 AI 使用历史（隐私/无统一导出口），
  Demo 用**一份真实感 mock 用法语料 + 确定性诊断引擎**呈现，并**诚实标注数据为模拟**。
  核心价值（行为诊断 → 针对性招式 → 可量化长进）用 before/after 对比可视化。
- 演示的是**我们分析出的创新切入点**（行为诊断优先的 AI 用法教练），**不是 AGINE Academy（课程优先）的克隆**。

## 6. 核心流程（3 分钟内可完成）
1. 进入「诊断台」：看到「我们分析了你最近 20 次和 AI 的协作」+ 基线上手指数 + 3 个关键指标
   （一次过率 / 平均返工轮数 / 放弃率），以及按「浪费返工轮次」排序的**失手签名卡片**。
2. 点某张失手卡「看这招怎么破」→ 进「招式库」：读该失手的人话微课 + 可复用招式（可复制的模板/清单）
   + **用你自己那次会话做的 before/after 复盘**（原始 prompt & 差结果 ↔ 应用招式后的 prompt & 好结果）。
3. 点「练这招 / 标记掌握」→ 到「长进」：看到一次过率↑、平均返工轮数↓、放弃率↓、上手指数↑ 的
   before/after 对比条随你掌握的招式实时变化；并诚实标注「哪个习惯仍需多练」。

## 7. 页面结构（<= 3 个主要页面）
- 页面 1「诊断台」：用法概览 + 基线指标 + 失手签名排序卡片（频次/平均多花轮次/严重度/示例会话）。
- 页面 2「招式库」：选中失手签名的微课 + 可复用招式 + 你自己会话的 before/after 复盘 + 「标记掌握」。
- 页面 3「长进」：before/after 价值可视化（生手基线 vs 应用招式后）+ 上手指数 + 诚实的「仍需练习」提示。

## 8. 关键交互
- 失手卡片点击 → 跳招式库并定位对应招式。
- 「标记掌握 / 取消掌握」→ 引擎重算全部指标，三页联动（诊断台的残余频次、长进页的对比条同步变化）。
- 掌握一招**不等于**该失手清零（习惯需要反复练）：引擎按「掌握后该签名出现率下降约 70%」建模，
  保证长进真实、上手指数不会瞬间满分（诚实）。
- 顶部标签切换三页；首屏一句话讲清「这是对你自己用法的体检，不是又一套课程」。

## 9. 使用的 mock 数据
- 一份 persona「林果」的 **20 条真实感 AI 协作会话**（`src/data/corpus.ts`）：每条含任务、领域、
  当时的 prompt、返工轮数、结果（满意/返工/放弃/将就）、命中的失手签名标记，以及用于复盘的「改后版本」。
- 一套**失手签名taxonomy**（7 类：目标含糊 / 没给范例 / 一股脑倒 context / 不给约束 / 首稿即收 /
  要事实不给依据 / 中途改需求），每类含人话解释、可复用招式、严重度权重。
- 全部为 mock，**不接真实后端 / LLM / 外部 API / 登录**；诊断由确定性引擎 `src/logic/engine.ts` 计算。

## 10. 明确不做什么
- 不做登录 / 用户系统。
- 不接数据库 / 支付 / 外部私钥 / 真实 API / 真实 LLM 调用。
- 不真的读取用户的 ChatGPT/Claude 历史（Demo 用 mock 语料，并明确标注为模拟）。
- 不做生产级后端；不超过 3 个主要页面；全部数据 mock。

## 11. 成功体验标准
- 用户 3 分钟内能说出：**解决什么**（诊断我自己反复犯的 AI 用法错误、教我系统性变好）、
  **怎么用**（看诊断 → 学招式 → 掌握后看长进）、**增量在哪**（诊断优先 + 从我自己的失手生成 + 抗过时，
  区别于课程优先的 AGINE 和只改单条的 prompt 优化器）。
- before/after 的上手指数与一次过率变化，让人直观感到「这真的能让我变好」。
