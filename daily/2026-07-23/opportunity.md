## 2. 候选机会（至少 3 个，均为创新切入点而非照抄）

### 候选 A：上手 Knack —— 从你自己的「失手」里，练出用 AI 的手感（AI 用法教练）
- 创新切入点：把「学会用 AI」从**课程优先（curriculum-first）**翻转为**行为诊断优先（behavior-first）**。
  不再给你一套通用剧情课，而是回看你自己最近 N 次和 AI 的真实协作，**诊断出你反复犯的「用法失手」签名**
  （目标含糊 / 没给范例 / 一股脑倒 context / 首稿即收 / 要事实不给依据…），按「浪费了多少返工轮次」排序，
  为你的 Top 失手生成**针对你的招式 + 用你自己那一次会话做的 before/after 复盘**，并用「上手指数」量化长进。
  因为每次从你**最新用法**重新诊断，所以**不会像课程那样过时**。
- 目标用户 / 场景：每天用 ChatGPT/Claude 干活、但「结果平庸、来回返工、说不清问题出在自己哪」的广大
  非开发者 + 进阶个人用户（远宽于 AGINE 偏开发者的受众）。
- 五维评分：痛点 4/5，机制新意 4/5，AI 核心度 4/5，一天可行性 4/5，启发价值 4/5 → 合计 **20/25**

### 候选 B：降配有据 Rightsize —— 给 agent 每一步「有据可依」的模型降配建议
- 创新切入点：顺着 Gemini「效率 + 垂直专用」信号，但不做又一个运行时路由网关（赛道拥挤）。而是对 agent
  流水线的**每个步骤**，用该步骤的一批留出样本 + 你的验收检查，量化「这一步换成更便宜/垂直的小模型，
  质量会不会掉」，把「盲目全用贵模型」的过度配置，变成**有回归证据的按步降配**。
- 目标用户 / 场景：为控成本发愁、又怕降配引发质量回归而不敢动的 agent/应用开发者。
- 五维评分：痛点 4/5，机制新意 3/5，AI 核心度 4/5，一天可行性 4/5，启发价值 4/5 → 合计 **19/25**
  （机制新意打折：评测+路由赛道拥挤，且与我 07-14「值当」的 econ 方向部分重叠。）

### 候选 C：分岔台 ForkLab —— 让 agent 在决策点分叉多路、对比后择优
- 创新切入点：box 让「从运行态 fork」变便宜，但缺的是**面向人的产品层**：在 agent 遇到岔路（用哪个方案/
  哪个库/哪种重构）时，一次分叉出 N 条支线并行尝试，跑完把结果**并排对比**（测试通过数/改动面/风险/耗时），
  由人一键择优、丢弃其余。今天 agent 是线性的，你永远看不到「没走的那条路」。
- 目标用户 / 场景：跑长任务、需要探索多方案的 agent 高级用户 / 开发者。
- 五维评分：痛点 3/5，机制新意 3/5，AI 核心度 3/5，一天可行性 4/5，启发价值 4/5 → 合计 **17/25**
  （受众窄 + 「parallel agents」概念拥挤，机制新意与 AI 核心度打折。）

> 另有一个**被我主动否决**的方向：照着 Kastra 做「确定性授权 PDP 面板」。虽然它是报告的 #1 信号，但
> (a) 治理/审批/审计是我近期已严重过度覆盖的家族，机制新意近乎为 0；(b) 极易沦为克隆；(c) 中间层易被
> IDE/CLI 原生审批挤掉。基于客观判断，**不选**。

## 3. 最终选择

- 选中：**候选 A — 上手 Knack**，总分 **20/25**
- 为什么选它（相对其他候选的优势）：
  1. **机会密度更高**：报告技术向拥挤且被大玩家挤压，2C 侧只有 AGINE 一个信号、还自曝两大软肋——
     我攻击的正是它没解决的部分，蓝海更宽。
  2. **痛点更宽更真**：「我用 AI 但结果平庸、不知问题在我哪」是数以亿计 AI 用户的高频真实痛，
     受众远宽于 AGINE 偏开发者的定位，也不像 B/C 那样局限于开发者。
  3. **机制新意最锋利**：行为诊断优先 + 从你自己的失手生成 + 抗过时，是对「AI 教育=内容供给」范式的
     真正翻转，而 B 与评测/路由/值当重叠、C 与 parallel-agents 重叠。
  4. **可诚实地做出可体验 Demo**：核心价值（行为诊断 → 针对性招式 → 可量化长进）可用一个确定性引擎在
     mock 语料上完整跑通并做 before/after 可视化，无需登录/外部 API。
  5. **符合独立判断**：主动避开了报告 #1 的治理信号（我已过度覆盖），把注意力放到被低估的 2C 侧。

## 4. 深入论证（针对选中机会）

1. 目标用户：**每天用 ChatGPT/Claude 处理真实工作**（写作、方案、分析、翻译、轻编码）、但自学到达
   瓶颈的**非开发者 + 进阶个人用户**——他们能用起来，却常「来回返工、结果只到 70 分、不知道差在哪」。
2. 核心痛点（真实、高频、可付费）：**「我用 AI 的结果平庸，但我不知道是我自己哪里用错了，更不知道怎么改。」**
   高频（几乎每天发生）、真实（不是伪需求）、有付费意愿证据（人们已经在买各种 prompt 课/教程）。
3. 现有产品为什么没解决好：
   - **课程类（AGINE Academy、各种 prompt 课）= 内容供给**：教的是课程里的通用任务，不是你自己的任务；
     且随模型迭代快速过时（报告已点名 AGINE 的这个软肋）；「学完不知会不会用」的迁移鸿沟没被填。
   - **Prompt 改写器 / 优化器 = 只改一条 prompt**：帮你把眼前这句改好，但不诊断你**反复犯**的习惯，
     不教你可迁移的元技能，你下次还错。
   - 没有产品把「**你自己的真实用法历史**」当作诊断对象，去回答「你到底反复错在哪、怎么系统性变好」。
4. 为什么适合做轻量 Demo：机会本质是「行为诊断 + 教练 + 长进可视化」，可以用一个**确定性引擎**在一份
   真实感的 mock 用法语料上跑通：诊断失手签名 → 排序 → 生成招式 + before/after 复盘 → 掌握招式后
   重新诊断、量化「上手指数」提升。纯前端、mock 数据、可 build，3 分钟看懂价值。

## 5. 不照抄声明（四项必填）

1. **创新切入点是什么**：把「学会用 AI」从**课程优先**翻转为**行为诊断优先**——以你自己的真实用法历史为
   诊断对象，找出你**反复犯**的用法失手签名，给出**针对你**的可迁移招式与可量化长进，并因持续从你的最新
   用法重新诊断而**抗过时**。
2. **解决的具体问题 / 场景**：广大 AI 用户「结果平庸却说不清问题出在自己哪、也不知怎么系统性改进」；
   正是报告点名 AGINE 的两大软肋（学完不知会不会用 / 内容易过时）所在的缺口。
3. **相对被参考产品的增量价值（差异点）**：
   - vs **AGINE Academy**：AGINE 是**课程/内容产品**（通用剧情闯关、养数字伙伴、卖订阅解锁课程），
     教的是课程任务；上手是**诊断+教练产品**（从你的失手签名生成、产出「你的个人招式库 + 长进曲线」），
     教你自己任务上的元技能，且抗过时；受众更宽（不止开发者）。
   - vs **Prompt 优化器**：那类只改单条 prompt；上手诊断的是**跨会话的重复习惯**，目标是让人**长期变好**。
4. **为什么这不是照抄 AGINE Academy（本质区别）**：数据来源不同（AGINE=预制课程 vs 上手=你自己的用法
   历史）、机制不同（内容闯关 vs 行为诊断+复盘）、产出不同（课程产物 vs 个人招式库+上手指数）、留存逻辑
   不同（订阅解锁课程 vs 持续诊断你最新用法、随你成长）。若做成「又一套 AI 课程」即视为失败，必须重做。
   （也与本实验室历史选题区隔：耳记=文档主动回忆、明镜=玻璃盒自适应导师、师承=专家红线教 agent、
   常青=文档新鲜度——上手的对象是**人本身用 AI 的技能**，机制是**从其行为失手签名做诊断教练**，均不同。）

## 6. 不确定性与风险

- 需求侧风险：付费意愿存疑——用户可能认为「用 AI 的技巧」不值得单独付费；教育类产品留存本就难。
  缓解：把价值锚定在「可量化的返工减少/一次过率提升」，让人看到省下的时间。
- 落地 / 技术风险：真实产品需读取用户的 AI 使用历史（隐私 + 各家 AI 无统一导出口）——这是最大工程/合规
  难点。Demo 阶段用 mock 语料 + 确定性引擎规避，但要**诚实标注数据为模拟**，不夸大「已能读你的历史」。
  另一风险：失手签名的自动识别真实场景要靠 LLM 分类，可能误判；产品需给「这是建议、可忽略」的余地。
- 我可能判断错的地方：也许多数用户对「变得更会用 AI」并无主动学习意愿（被动够用即可），那么这更像
  「有用但难变现」的工具；也可能大厂（OpenAI/Anthropic）会把这类「用法体检」直接内建进官方产品。

- 是否达到 16/25 门槛：**是**
