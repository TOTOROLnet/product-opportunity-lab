# 知时 Aptly — 该出声时才出声的环境记忆层

> 2026-07-20 每日产品机会 Demo。基于 product-hunt-radar `2026-07-20` 报告（Rewisp「环境记忆经 MCP 供给 agent」+「MCP 固化为记忆/数据/技能标准接口」趋势）做出的**独立创新切入点**，非任何产品的克隆。

## 这是什么

环境记忆产品（如今日雷达里的 **Rewisp**）把你看过的屏幕文本存下来、供你 **PULL 提问**。但最痛的场景是——**你根本没意识到自己忘了**，PULL 解不了这个悖论；而朴素的"主动提醒"又必然沦为噪音被你关掉。

**知时 Aptly** 不做记忆存储，只做垫在**任意环境记忆源**之上的 **「主动召回决策层」**：

- 用 `相关度 × 时效 × 可行动性 × 新鲜度` 打分；
- 再用**滚动「打扰预算」**闸门 + **去重** + **过时抑制**，决定此刻是把某条旧记忆推到你眼前、还是**保持安静**。

一句话：把"提醒一切"变成"**该提醒时才提醒**"。

## 怎么证明价值（可视化）

一个**确定性引擎**（`src/logic/engine.ts`）在同一条脚本化的「一个上午」工作流上，用**朴素推送**与**知时**两种策略回放，指标全部**实测**而非编造：

| 指标 | 朴素推送 | 知时 Aptly |
| --- | --- | --- |
| 总打扰 | 8 | **4** |
| 精确率 | 38% | **100%** |
| 关键时刻覆盖 | 3/4（漏"退款 API 参数变更"） | **4/4** |
| 过时/有害推送 | 2 | **0** |
| 正确沉默 | 1/4 | **4/4** |

## 三个页面

1. **工作台（时间线回放）**：切换朴素/知时，播放/单步走完一个上午，看每个"现在"推了什么、为什么，以及累计指标。
2. **决策透视（玻璃盒）**：每个"现在"对每条候选记忆的相关度、综合分、决策与原因全部摊开，可按记忆筛选。
3. **对比**：朴素 vs 知时 并排，实测指标 + 每步行为差异。

## 本地运行

```bash
npm install
npm run dev      # 开发
npm run build    # 产出 dist/（vite.config 已设 base:'./'，可部署到任意 Pages 子目录）
npm run preview  # 预览 build 产物
```

## 技术与边界

- 技术栈：Vite + React + TypeScript；`vite.config.ts` 设 `base:'./'`。
- **纯前端静态 Demo**：全部数据为 mock、时间线脚本化；**不接** 后端 / 数据库 / 登录 / 支付 / 外部 API / 真实 LLM / 真实 agent。
- 相关度/可行动性用**确定性代理函数**模拟（真实产品需真数据训练的召回模型）；本 Demo 只证明"机制成立与价值形态"，不代表真实召回质量。

## 与 Rewisp 的区别（非克隆）

- Rewisp = 记忆**存储 + PULL 查询 + 单规则提醒（承诺追踪）**；
- 知时 = 垫在其上的 **PUSH 决策层**（把 Rewisp 唯一的硬规则 PUSH，泛化为有节制的通用主动召回）。
- 两者是**互补的两层**：去掉任一层，另一层仍成立。知时甚至可以直接消费 Rewisp 暴露的 MCP 只读记忆源。
