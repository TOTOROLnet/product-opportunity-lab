## 2. 候选机会（至少 3 个，均为创新切入点而非照抄）

### 候选 A：知时 Aptly —「该出声时才出声」的环境记忆决策层

- 创新切入点：不做记忆存储（那是 Rewisp/Rewind 的战场），而做垫在**任意环境记忆源之上**的**主动出手（PUSH）决策层**：用 `相关度 × 时效衰减 × 可行动性 × 新鲜度(去重/去过时)` 打分，并用一个**滚动"打扰预算"**闸门决定——此刻是把某条旧记忆推到你眼前，还是**保持安静**。把 Rewisp 的"承诺追踪"从单一硬规则，泛化为**有节制的通用主动召回**。
- 目标用户 / 场景：在几十个 App/网页间高频切换的知识工作者/独立开发者/运营——"20 分钟前在 Slack 看到 Acme deadline 提前到周四，现在正给 Acme 回信却完全没想起来"。
- 五维评分：痛点 4/5，机制新意 4/5，AI 核心 4/5，一天可行性 5/5，启发价值 4/5 → 合计 **21/25**

### 候选 B：过境 CrossGuard — 环境记忆→云/agent 的"出境预览 + 可逆脱敏"层

- 创新切入点：报告明确点出 Rewisp 的隐患——模型逐级回退会把**本地私密屏幕记忆升级发往 Gemini/Claude/ChatGPT**，且 MCP 只读让任意客户端可查全部屏幕历史。切入点：在"离开本机/流入 agent 上下文"的**边界**做**出境预览**——自动标出即将外传的敏感片段（密码/客户名/薪资/病历），一键**可逆令牌化**脱敏，agent 拿到脱敏但仍可用的上下文，回答再本地**回填**。
- 目标用户 / 场景：用云端 agent 但记忆里混着隐私数据的个人与小团队。
- 五维评分：痛点 4/5，机制新意 3/5，AI 核心 3/5，一天可行性 4/5，启发价值 4/5 → 合计 **18/25**
- （note：这是"边界管控/DLP"味道，属于我 07-03~07-13 严重过度的 governance/审批控制面家族，即使分数不低也应主动避让以保持机制多样性。）

### 候选 C：定论 Settled — 环境记忆的"时间感知定论/矛盾消解"层

- 创新切入点：被动记忆流里同一事实会被反复覆盖（旧 SOP ¥500 → 新 SOP ¥1000）。agent 朴素检索会取回**过时/矛盾**片段。切入点：查询时不返回原始片段，而返回**按时间/来源排序、显式标注矛盾**的"当前定论 + 佐证 + 冲突项"。
- 目标用户 / 场景：让 agent 消费环境记忆而怕它引用陈旧信息的人。
- 五维评分：痛点 4/5，机制新意 3/5，AI 核心 4/5，一天可行性 4/5，启发价值 3/5 → 合计 **18/25**
- （note：与我 07-17「常青 Evergreen」知识新鲜度/自愈家族高度重叠，有自我重复风险，主动降权。）

## 3. 最终选择

- 选中：**候选 A — 知时 Aptly**，总分 **21/25**
- 为什么选它（相对其他候选的优势）：
  1. **最锋利地打中报告没点破的真痛点**：Rewisp 是 PULL，而环境记忆最痛的是"没意识到自己忘了"——只有 PUSH 能解，但历史上 PUSH 都死于噪音。"有节制的 PUSH"是这个赛道真正缺的判断层。
  2. **机制最新颖且非我过往家族**：B 落回我严重过度的 governance/控制面家族；C 与我 07-17 常青重叠。A 是全新的 **proactive-recall / 注意力经济学** 家族（"克制"本身成为产品）。
  3. **可诚实演示**：确定性引擎在同一条脚本化"一早工作流"上跑「朴素推送 vs 知时」双模式，打扰次数/精确率/漏关键/保持安静等指标由引擎实测而非编造。
  4. **补 2C 平衡**：一个消费/prosumer 生产力产品，且不因报告说"今日无 2C"而放弃独立判断。

## 4. 深入论证（针对选中机会）

1. 目标用户：日均在 20+ App/标签页间切换、"看过就忘、忘了还不自知"的知识工作者与独立开发者（先服务个人/prosumer，再以 MCP 接入 agent）。
2. 核心痛点（真实、高频、可付费）：**你在错误的时刻缺了一条你其实已经看过的信息**——给 Acme 回信时忘了它 deadline 已提前；改退款代码时忘了 API 参数刚改名；答应周五发报价却漏了。这类"本可避免的疏漏"高频且有真实代价，prosumer 愿为"关键时刻替我记得"付费。
3. 现有产品为什么没解决好：
   - Rewind/Recall：录屏重、踩隐私，且是 PULL；
   - Rewisp：更干净（纯文本+本地+MCP），但仍以 PULL 为主，唯一 PUSH（承诺追踪）是单规则；
   - 系统级通知/待办：靠你**手动**设定，等于要你先记得——回到同一个悖论；
   - 一切"主动提醒"类产品的通病：**没有克制**，相关度低也推、推完还重复推，最终被用户关掉。
4. 为什么适合做轻量 Demo：核心是一个**决策引擎**（打分 + 预算闸门 + 去重/去过时），可在纯前端用脚本化时间线确定性回放，用 before/after + 玻璃盒把"克制的价值"直接可视化，无需任何后端/真实模型/隐私数据。

## 5. 不照抄声明（四项必填）

1. 创新切入点是什么：一个**主动召回的决策/克制层**——`相关度×时效×可行动性×新鲜度` 打分 + **滚动打扰预算**闸门 + 去重/去过时抑制 + "默认沉默"立场；它**消费**任意环境记忆源（包括 Rewisp 的 MCP），自身不做记忆存储。
2. 解决的具体问题 / 场景：解决 PULL 解不了的"不知道自己忘了"，同时避免朴素 PUSH 的"噪音刷屏"——在**正确的时刻**只推**该推的那一两条**，其余闭嘴。
3. 相对被参考产品的增量价值（差异点）：Rewisp = 记忆**存储 + PULL 查询 + 单规则提醒**；知时 = 垫在其上的**PUSH 决策层**（把"提醒一切"变成"该提醒时才提醒"）。二者是**互补的两层**：去掉任一层另一层仍成立。
4. 为什么这不是照抄 Rewisp（本质区别）：Rewisp 的产品是"存下来、你来问"；知时的产品是"**判断此刻该不该由我主动开口**"。前者是记忆基础设施，后者是注意力/时机决策——不同层、不同价值主张，绝非克隆。（同理也非 Google Now 式旧 assistant 的复刻：它们死于无克制，知时的**核心恰恰是那套克制机制**。）

## 6. 不确定性与风险

- 需求侧风险（**最大风险**）：主动提醒是 PMF 的重灾区。即使有克制，用户对"被打扰"的容忍度极低；一次误报可能就让人关掉。缓解思路：默认极度保守（宁可漏也不吵）、预算可调、每条推送都给"为什么现在推"的理由并可一键"以后别再这样推"。
- 落地/技术风险：真实相关度/可行动性模型需要真数据训练；本 Demo 用确定性代理函数模拟，只证明"机制成立与价值形态"，不代表真实召回质量。
- 我可能判断错的地方：也许对多数人来说 PULL 已够用，PUSH 的边际价值撑不起独立产品，而更应作为 Rewisp 这类记忆产品的一个功能存在——若如此，本机会更适合作为"记忆产品的判断层特性"而非独立创业标的。

- 是否达到 16/25 门槛：**是**
