# Pane（明窗）— 生成式 UI 的可信层 · Demo

> 2026-07-10 每日产品机会循环产出。纯前端静态原型，全部数据为 **mock**，
> 不接真实模型 / 支付 / 账户 / 数据库 / 外部 API。

## 这是什么

消费级 AI 正在「逃离聊天框」——用**现场生成的可交互界面**取代长文本（如 Product Hunt 上的
Monogram、Google 的 generative UI）。但生成式 UI 引入了聊天没有的新问题：**界面本身是模型输出**，
因此模型可以编造界面里的数据与操作。一个编造的价格放进真实感的价签里，比放进一段文字里更具欺骗性。

**Pane** 给生成式 UI 加一层「可信玻璃」：让每一个被模型生成的界面元素都能回答三个问题——

- 这个数字是**真的**吗？（溯源：来源 / 抓取时间 / 判定理由）
- 这个按钮**真的会做它说的事**吗？（预检：明文效果；真花钱需确认；占位按钮直接标注）
- 这个界面**几成是模型编的**？（度量：界面级「真实度」仪表）

## 怎么体验

打开后：

1. 顶部在 **「裸生成式 UI」** 与 **「接入 Pane 明窗后」** 之间切换，看同一张界面的差别。
2. 切换三个场景：
   - **订东京机票**：一个被编造的「最低价 ¥1,280」+ 一个看似能下单、其实**未绑定真实能力的占位按钮**。
   - **账单分析**：一个被算错、且与已核实明细自相矛盾的「本月支出」总额 + 一个**真花钱**的分期按钮。
   - **食谱换算**（对照组）：全部已核实，Pane 为其**整体背书**——证明它不是「狼来了」报警器。
3. Pane 模式下：点任意带徽标的元素看**溯源**；点动作按钮先做**明文预检**。

## 本地运行

```bash
npm install
npm run dev      # 本地开发，默认 http://localhost:5173
npm run build    # 产物输出到 dist/
npm run preview  # 预览已 build 的产物
```

## 技术栈

- Vite + React + TypeScript
- `vite.config.ts` 设 `base: './'`（相对路径），可部署到任意 GitHub Pages 子目录。

## 目录

- `src/types.ts` — 核心数据模型（元素 / grounding / provenance / action）。
- `src/data/scenarios.ts` — 3 个 mock 场景（含每个元素的预置 grounding 标注）。
- `src/logic/grounding.ts` — 真实度聚合、动作预检、裸模式对照结果等纯逻辑。
- `src/components/` — 场景切换、模式开关、生成卡片、徽标、真实度仪表、溯源/预检弹层、原理说明。

## 诚实性声明（重要）

Demo 中「模型生成的界面」与「每个元素的 grounding 判定」都是**预先标注的 mock**，用于演示 Pane 的
价值主张，**不代表任何真实模型推理、真实价格或真实账单**。真实产品里，这些元数据应由生成式 UI
框架在生成时暴露（每个组件绑定的数据源与工具），再由启发式 / 模型判定得出。
