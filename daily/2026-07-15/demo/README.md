# 耳记 · Earmark — 主动回忆式音频学习台（Demo）

> 把任意长文档变成"边听边被追问"的音频课。**听不是终点，而是主动回忆的载体。**
> 别人做"更好听的播客"，耳记做"听完真的记住"。

这是 2026-07-15 每日产品机会循环的纯前端 Demo（Vite + React + TypeScript）。
数据全部为 mock，不接后端 / 数据库 / 支付 / 登录 / 外部 API / 私钥。
语音使用**浏览器原生 `window.speechSynthesis`**（非外部服务、不上传数据），
环境不支持时自动降级为"进度条 + 文字高亮"，不影响体验闭环。

## 它解决什么问题

用户花通勤/碎片时间"听掉"长报告、教材、论文，却**留存极低、说不清薄弱点**。
被动听（TTS / 播客 / NotebookLM 音频概览）是线性、无反馈的——听完≈没听。
耳记把这段被动收听时间，转化为**可测量、可巩固**的学习成果。

## 核心体验（3 分钟闭环）

1. **选材台**：从 mock 资料库选一份材料（科普 / 商业备忘 / 教材），
   看到 AI 把它切成的"概念单元 + 每单元追问数 + 预计时长"。
2. **收听台**：逐句朗读/高亮当前概念；播到概念边界会**自动暂停发问**（选择题 + 你的把握度），
   答完立刻反馈 + 解释，右侧"保留度地图"实时更新。可倍速、可"跳到追问"。
3. **回顾台**：看到**逐概念保留度地图**、**被动听 vs 主动回忆的估算留存对比（before/after）**、
   以及**自适应"60 秒重听队列"**（最薄弱、以及"自信地答错"的概念排最前，可单概念回炉重测）。

## 相对 VocalVia / NotebookLM 的差异（不照抄声明）

- 它们优化"产出更好听/可编辑的音频"（**消费侧**指标）；耳记优化"**听完的留存与薄弱点定位**"（学习侧指标）。
- 同样从文档出发、同样产出音频，但**目标函数不同**：终点是"用户对材料的可测量掌握"，不是"一段能听的播客"。
- 独有的学习闭环：被打断的主动回忆、保留度地图、"危险的自信"（自信地答错）识别、自适应重听队列。

## 本地运行

```bash
npm install
npm run dev      # 本地开发预览
npm run build    # 产出 dist/（tsc -b && vite build）
npm run preview  # 预览已构建产物
```

- `vite.config.ts` 已设 `base: './'`，可部署到任意 GitHub Pages 子目录。

## 目录结构

```
src/
  App.tsx                 三视图状态机（选材 / 收听 / 回顾）
  types.ts                领域类型
  data/library.ts         mock 语料（3 份文档，含概念单元 + 苏格拉底追问）
  logic/mastery.ts        纯逻辑：作答×把握度→掌握分、留存估算、重听队列
  audio/speech.ts         浏览器 speechSynthesis 封装（不支持则降级）
  components/
    LibraryView.tsx       选材台
    PlayerView.tsx        收听台（核心：被打断的主动回忆）
    RecapView.tsx         回顾台（保留度地图 / before-after / 重听队列）
    shared.tsx            进度条 / 状态标签 / 分数环等小组件
  index.css               全部样式
```

## 明确不做（Demo 边界）

- 不做真实文档解析或真实 LLM 生成（概念单元与追问为预置 mock）。
- 不接数据库 / 支付 / 登录 / 外部 API / 真实 TTS 云服务。
- 不做生产级后端；主要页面 ≤ 3 个。

留存对比中的"被动听基线"为教育学经验值，用于示意，非精确测量数据。
