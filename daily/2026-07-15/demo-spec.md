# Demo 设计说明 — 2026-07-15

## 1. Demo 名称
耳记（Earmark）— 主动回忆式音频学习台

## 2. 一句话定位
把任意长文档变成"边听边被苏格拉底式追问"的音频课：听不再是终点，而是主动回忆的载体——听完能量化你到底记住了什么、薄弱点在哪、并自动排出 60 秒重听队列。

## 3. 目标用户
要把长报告 / 教材 / 论文"听掉"的学生、考证与自学者、要吃行业报告的知识工作者。他们已在用 TTS / NotebookLM 听材料，但"听完等于没听"。

## 4. 使用场景
通勤或碎片时间，想省眼睛又想真学会：选一份材料 → 生成音频课 → 边听边被追问 → 看到自己的保留度地图 → 拿到"薄弱概念重听队列"。

## 5. Demo 策略分型（重要）

- 本机会属于：**■ 可视化/交互类产品**（消费级学习体验，核心价值靠交互闭环说清）
- 采用的演示方式：直接做**可点击交互 Demo**。用 mock 语料（预切好的概念单元 + 预生成的苏格拉底追问 + 干扰项 + 解释）驱动完整闭环；音频用**浏览器原生 `window.speechSynthesis`**（非外部 API、无需 key、无 secrets）让"听"有真实感，环境不支持时自动降级为"模拟播放进度条 + 脚本高亮"，不影响 build 与核心体验。
- 演示的是**我们分析出的创新切入点**（把音频重构为主动回忆的载体），不是 VocalVia / NotebookLM 的克隆。

## 6. 核心流程
（用户从进入到"看懂价值"的主路径，3 分钟内可完成）
1. **选材**：从 mock 资料库选一份文档（如一篇科普 / 一份商业备忘 / 一段教材），看到 AI 把它切成的"概念单元 + 每单元的追问数 + 预计时长"，点"开始主动回忆式收听"。
2. **收听 + 被追问**：进入播放器，当前概念单元的脚本逐句高亮、语音播报；播到概念边界会**自动暂停弹出回忆卡**（选择题 + 我的把握度）；答完立刻给对/错与解释，并更新右侧"保留度地图"。答错或低把握 → 该概念被标记进重听队列。
3. **回顾**：全部听完后进入回顾页，看到**逐概念保留度地图**、**被动听 vs 主动回忆的估算留存对比（before/after）**、以及**自适应生成的"60 秒重听队列"**；可一键把某个薄弱概念"重听并再测"。

## 7. 页面结构（<= 3 个主要页面）
- 页面 1：**选材台 Library** — mock 文档卡片、AI 拆解出的概念单元预览、开始按钮。
- 页面 2：**收听台 Player** — 音频/进度、脚本高亮、概念边界处的"主动回忆卡"、实时保留度侧栏。
- 页面 3：**回顾台 Recap** — 保留度地图、before/after 留存对比、重听队列（可回到 Player 重测单个概念）。

## 8. 关键交互
- 播放/暂停、逐概念进度；到边界自动暂停并弹回忆卡（模拟"被打断的主动回忆"）。
- 回忆卡：多选一 + 把握度（高/中/低）；提交后即时反馈 + 解释；据"是否答对 × 把握度"更新该概念 mastery 分。
- 保留度地图：每个概念一个状态块（已掌握 / 模糊 / 薄弱），随作答实时变化。
- before/after：用当次真实作答估算"主动回忆留存"，对比一个固定的"被动听基线"，直观显示增量。
- 重听队列：把薄弱概念排成短时"60 秒重听"，支持单概念回炉重测。

## 9. 使用的 mock 数据
（全部为 mock，不接真实后端 / 外部 API / 模型）
- `src/data/library.ts`：2–3 份 mock 文档，每份含：标题、来源类型、若干**概念单元**（unitId、标题、音频脚本文本、1–2 道回忆题：题干 + 选项 + 正确项 + 解释）。
- 语音：运行时调用浏览器 `speechSynthesis`（可用则朗读脚本，不可用则纯进度条 + 文字高亮）。
- 掌握度/留存：`src/logic/mastery.ts` 用纯函数把"作答正确性 × 把握度"映射为概念 mastery 与整份材料的估算留存；被动听基线为固定常量（标注为教育学经验值，非精确数据）。

## 10. 明确不做什么
- 不做登录 / 用户系统。
- 不接数据库 / 支付 / 外部私钥 / 真实模型 API / 真实 TTS 云服务。
- 不做真实文档解析或真实 LLM 生成（概念单元与追问均为预置 mock）。
- 不做生产级后端；不超过 3 个主要页面。

## 11. 成功体验标准
- 用户 3 分钟内能说出"它解决什么问题（听完能记住）、怎么用（听→被追问→看保留度→重听）、增量在哪（对比 VocalVia/NotebookLM 的消费侧，它优化的是留存）"。
- 至少完成一次"听 → 被打断追问 → 看到保留度地图变化 → 拿到重听队列"的闭环。
- before/after 对比让人直观感到"主动回忆比被动听多记住多少"。
