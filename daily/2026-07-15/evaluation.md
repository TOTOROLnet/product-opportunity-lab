# 自动体验 / 自测报告 — 2026-07-15

> 以 AI 产品经理 + 目标用户双重视角，对本次 Demo 做一次体验检查。结论只能是 PASS / PARTIAL / FAIL。

## 1. 构建结果
- `npm install`：成功（67 packages）。
- `npm run build`：成功（`tsc -b && vite build`，首轮通过，失败修复轮次：0/3）。产物 `dist/index.html` 0.57 kB、JS 160.51 kB（gzip 56.30 kB）、CSS 11.95 kB。
- smoke 检查（`scripts/validate_demo.sh`）：通过——dist 首屏非空、含 `id="root"` 挂载节点、有 1 个 JS bundle。

## 2. 可用性检查
- 本地 `npm run preview`（localhost:4173）返回 200，页面正常加载。
- 首屏不空白：Hero"耳记 · Earmark" + 被动听/主动回忆对比条 + 3 张材料卡片正常渲染（见 `screenshots/01-library.webp`）。
- 用计算机操作子代理真机走查了完整主路径，无控制台报错、无断点。

## 3. 核心流程是否闭合
**闭合。** 实测完整走通：选材 → 收听（脚本高亮 + 进度条 + 可"跳到追问"/倍速/朗读开关）→ 概念边界自动暂停并弹回忆卡 → 选答案 + 选把握度 → 即时反馈 + 解释 + 该记住的一句话 → 右侧保留度实时更新 → 全部听完进入回顾 → 保留度地图 + before/after 留存对比 + 60 秒重听队列 → 点"重听并再测"回到单概念小节、答完回到回顾且分数更新（实测从 52% 更新到 78%）。

## 4. 首屏是否讲清产品价值
清楚。Hero 一句话点明"听不是终点，而是主动回忆的载体"，并用"被动听（听完≈没听）→ 主动回忆听（可测量留存 + 重听队列）"的 before/after 卡片，3 秒内让人 get 到差异化定位，而不是"又一个文档转播客"。

## 5. 交互是否有断点
未发现断点。倍速切换、朗读开关（环境不支持时自动降级为文字模式）、跳到追问、单概念重听等分支均正常；`speechSynthesis` 以定时器为进度真相、语音仅作增强，规避了 TTS 不可用导致卡流程的风险。

## 6. 是否只是概念包装（自嗨检测）
- 是否引用了报告中的具体信号：**是**。直接源于报告板块 B 的 VocalVia（文档→可编辑多角色音频）及其"review-before-credits"卖点，并显式质疑/修正了报告"2C 不值得跟进"的隐含结论。
- 是否明确了创新切入点与增量价值：**是**。切入点=把品类从"内容消费"重构为"主动回忆式学习"；增量=从"音频好不好听"转向"听完的留存 + 薄弱点定位 + 自适应重听"，并独有"危险的自信（自信地答错）"识别。
- 是否是某产品的照抄：**否**。VocalVia/NotebookLM 的终点是"产出可听/可编辑音频"，耳记的终点是"用户对材料的可测量掌握"，目标函数、核心交互（被打断的主动回忆）、留存指标、重听调度均不同。

## 7. Demo 的最大问题
- 概念单元与追问是**预置 mock**：真实产品最难的是"从任意文档自动切概念 + 生成高质量苏格拉底追问 + 给自由作答评分"，Demo 用选择题 + 固定语料回避了这块，价值验证仍偏"体验说服"而非"生成质量验证"。
- 留存对比中的"被动听基线（22%）"是教育学经验值、非精确测量，已在页面注明，但仍需避免被误读为实测数据。
- 语音自然度依赖系统 TTS，音质无法代表真实产品体验。

## 8. 是否建议用户人工体验
建议。核心"被打断的主动回忆 + 保留度地图 + 自信地答错识别"用交互体验最直观；用一份材料完整走一遍（含故意"自信地答错"）约 2–3 分钟即可感受差异化价值。

## 9. 下一步建议
- 接真实文档解析 + LLM 自动切概念/生成追问/给自由作答评分，验证生成质量与延迟。
- 加入间隔重复（spaced repetition）调度：把"重听队列"从单次会话扩展到跨天的记忆巩固曲线。
- 增加"背景/专注"两种模式：解决"我就想放松地听"与"我要认真学"的预期冲突（对应 opportunity 的需求侧风险）。
- 用真实用户的留存/复购数据证伪或证实"主动回忆是否伪需求"。
