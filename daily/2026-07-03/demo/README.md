# Contextlens — Agent 上下文窗口 "X 光片"（Demo）

纯前端交互 Demo，全部数据为 **mock**，不接任何后端 / 数据库 / 支付 / 登录 / 外部 API / 真实 LLM。

一句话定位：让 Agent 的"脑内上下文"变得可见——逐步透视上下文窗口里装了什么、什么被截断/驱逐/淹没，
以及它如何直接导致了错误或危险行为；再一键 Pin / 压缩看修复对比。

## 三个视图

1. **Run X 光**：执行时间线（逐步）＋该步"上下文窗口 X 光"（各块按 token 占比堆叠，被驱逐/截断的块置灰并标注原因）。
2. **修复对比**：同一第 6 步决策时刻，`基线`（策略被驱逐 → 危险放款）对比 `修复`（Pin 策略 + 压缩工具 → 安全转审批）。
3. **X 光报告**：窗口健康成绩单（驱逐事件 / 缺失关键约束 / 冗余工具 tokens）＋修复建议＋结论。

## 演示脚本（3 分钟）

1. 默认在 `基线` 场景、时间线停在 **第 4 步**：看到关键『退款政策』被静默驱逐（置灰、删除线、原因）。
2. 点 **第 6 步**：看到因策略不在窗口，Agent 直接放款 $900（危险）。
3. 顶部切到 **修复对比**：右侧 Pin 策略 + 压缩工具后，第 6 步策略仍在窗口 → 改为"请求人工审批"（安全）。
4. 切到 **X 光报告**：看健康成绩单与修复建议。

## 运行

```bash
npm install
npm run dev       # 本地开发预览
npm run build     # 类型检查 + 构建到 dist/
npm run preview   # 预览构建产物
```

## 技术

Vite + React + TypeScript。`vite.config.ts` 使用 `base: './'`，构建产物可部署到任意子路径
（如 GitHub Pages 的 `/product-opportunity-lab/2026-07-03/`）。

## 说明与边界

- 所有 token 数、工具输出、驱逐事件均为**示意 mock**，仅用于讲清"窗口装配 → 驱逐/淹没 → 行为回归"的因果。
- 真实产品需从各 Agent 框架采集"每步实际送入模型的完整上下文"，这是真实落地的最大工程挑战，Demo 用 mock 规避。
