# Rootline（根脉）— Demo

> 模型内多智能体调用的「结论溯源 + 压缩失真」审计透镜。
> product-opportunity-lab · 2026-07-11 · 纯前端静态 Demo（Vite + React + TypeScript）。

## 这是什么

2026-07-11，OpenAI GPT-5.6（`multi_agent` beta）与 Meta Muse Spark 1.1 同日把「多智能体编排」
从应用脚手架**下沉为模型 / API 原语**：根代理在**一次调用内** spawn 一棵子代理树，彼此
message / wait，并由服务端**自动压缩（compaction）**分别管理各代理上下文。

能力上移的同时，过程变成**原子黑盒**，且压缩是**有损**的——叶子代理只把「摘要」上报，
父代理再压缩再上报，根代理最终基于「**摘要的摘要**」作答。一个本可推翻结论的关键约束 /
数字 / 前提，可能在深层子树被悄悄压掉，而根代理**从未见过它**，却照样自信作答。

**Rootline** 给最终答案里的每一条承重结论追一条穿过压缩层的**证据根脉**，判定：

- **存续**：支撑事实穿过压缩层保持完整；
- **失真**：承重细节被压缩丢弃 / 弱化，结论具误导性；
- **断链**：结论是根代理的综合产物，其支撑证据未在压缩链中存续。

## 怎么用（3 分钟看懂）

1. 默认是「**裸答案**」态：看模型给的自信最终答案（看起来可直接执行）。
2. 点「**开启根脉透镜**」：中央出现子代理树（被压缩的节点带 ⚠ 徽标），右侧列出承重结论并标注
   存续 / 失真 / 断链，顶部「真相计」汇总是否可照此执行。
3. 点任意一条**结论** → 树上高亮它的**根脉路径**，右侧展开根脉逐跳与每跳被压缩丢弃的承重事实。
4. 点树上任意**节点** → 看它**压缩前后**的原始发现 vs 上报摘要 vs 被丢弃事实。
5. 切换到「**文献综述（对照组）**」：所有事实都存续、透镜全绿背书——证明它不是「逢结论必报警」。

## 本地运行

```bash
npm install
npm run dev      # 本地开发（默认 http://localhost:5173 ）
npm run build    # 生产构建，产物在 dist/
npm run preview  # 预览构建产物
```

## 技术说明

- Vite + React 18 + TypeScript；`vite.config.ts` 设 `base: './'`，可部署到任意 GitHub Pages 子目录。
- 目录：
  - `src/types.ts`：数据结构（子代理树节点 / 承重结论 / 场景）。
  - `src/data/scenarios.ts`：3 套**带标注的 mock 场景**（差旅预订 / 事故根因 / 文献综述对照组）。
  - `src/logic/rootline.ts`：根脉计算（路径、逐跳压缩损失、场景汇总判定）。
  - `src/components/*`：场景切换、裸/透镜切换、真相计、子代理树、最终答案、根脉溯源面板、方法论。

## 边界（明确不做）

- ⚠️ **全部为人工构造的 mock 数据**，用于证明产品价值主张。
- **不调用任何真实模型（OpenAI / Meta）**，不接后端 / 数据库 / 支付 / 密钥 / 登录 / 外部 API。
- 不复刻编排 / 工作台产品，不做真实 trace 采集。真实落地需从模型侧拿到子代理过程数据与压缩前后差异
  （见 `../opportunity.md` §6 风险讨论）。
