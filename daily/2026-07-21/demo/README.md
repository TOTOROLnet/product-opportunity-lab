# 信任阶梯 TrustLadder — Agent 写权限的渐进授权层

> 2026-07-21 每日产品机会 Demo。基于 product-hunt-radar `2026-07-21` 报告的
> 「自主 ops agent 按职能垂直落地 · 审批+审计 成核心机制」趋势（Rex / Nautis / Lunen / Backdrop / Creed 等，均为**报告事实**）
> 做出的**独立创新切入点**，**非任何产品的克隆**。

## 这是什么

今天几乎所有「运行公司的自主 ops agent」都靠一句话建立信任：**读放行 / 写审批 / 全程审计**。
但它们把「审批」做成了**逐条二元闸门**，把「审计」做成了**只写不读的墓地**。于是随着 agent 变多，
**人成了审批瓶颈**——要么盖橡皮章（治理沦为表演），要么什么都拦（自主性价值蒸发）。
报告说「自主执行的信任要一格一格挣」，却没有产品定义过「一格是什么、怎么挣、被辜负后怎么掉」。

**信任阶梯 TrustLadder** 不做又一个 agent，而是垫在**任意 ops agent 之上的一层写权限治理面**：

- **把你的批准/拒绝决策实时编译成可撤销的分级自动放行策略**：对相似动作聚类，
  当你一致批准到门槛，主动建议把某一类「低可逆·低风险」动作**提升为自动放行**；
- **信任阶（trust rung）状态机**：一致批准 +1 格，到门槛可自动；任一自动放行动作被反转/投诉，
  该类信任**立即 −1 格并暂停自动**——挣得慢、掉得快、全程可审计、可一键收回；
- **安全红线**：高风险 / 不可逆类目（退款大额、信用冻结、法务、批量、生产部署）**锁定为始终人工**，
  无论证据多少都不自动。

一句话：**把「每条都要人点」变成「重复的养成自动、人只看真正险的那几条」，且信任可度量、可回退。**

## 怎么证明价值（可视化）

一个**确定性引擎**（`src/logic/engine.ts`）在同一条脚本化的「一个上午」多 agent 动作流
（27 条，`src/data/scenario.ts`）上，用**全人工闸门**与**信任阶梯**两种策略回放，指标**实测**而非编造：

| 指标 | 全人工闸门 | 信任阶梯 |
| --- | --- | --- |
| 需人工审阅 | 27 次 | **18 次（−33%）** |
| 模拟清空耗时 | 40.5 分钟 | **27 分钟** |
| 自动放行覆盖率 | 0% | **~43%（仅第一天）** |
| 高风险/不可逆被无人工放行（安全红线） | 0 | **0** |
| 可逆错误被自动漏过（反转自愈） | 0 | 1（当场降级） |

> 诚实地说：信任阶梯用「1 条**可逆**且被即时反转自愈的小错误」换来了大幅降载，
> 但对「不可逆动作」永远坚持人工——省力 ≠ 失控。且被养成的策略**跨天累积**，覆盖率会随天数与 agent 数复利。

## 三个页面

1. **审阅台（Cockpit）**：逐条处置 agent 动作（批准/改后批准/拒绝/升级），
   到门槛时弹出「提升为自动放行」建议；提升后同类自动流过、计数器实时下降；
   脚本进行到「反转事件」会红色横幅提示并把该类信任降一格。也可用「推荐下一步」一键推进。
2. **信任阶梯（Ladder）**：所有类目的信任阶（0–5 格）总览、当前生效的可撤销策略（可一键「收回自动」）、
   以及「一格」语义与安全红线说明。
3. **价值对比（Compare）**：全人工闸门 vs 信任阶梯，在同一脚本上的实测指标并排 + 安全红线/诚实代价/复利三点结论。

## 技术与边界

- 技术栈：Vite + React + TypeScript；`vite.config.ts` 设 `base:'./'`。
- **纯前端静态 Demo**：全部数据为 mock、时间线脚本化；**不接** 后端 / 数据库 / 登录 / 支付 / 外部 API / 真实 LLM / 真实 agent / 真实 MCP。
- 「可逆性估计」「相似动作聚类」用**确定性代理**模拟（真实产品需真数据与安全的 agent 批准 hook，很可能经 MCP 的 write-approval 协议）；本 Demo 只证明「机制成立与价值形态」，不代表真实归纳质量。

## 与被参考产品的区别（非克隆）

- **Rex / Nautis** = 垂直 ops agent，可逆性**硬编码**在自己产品里；TrustLadder = **横切所有 agent** 的控制面，
  策略**从你的行为学出来**并随偏好进化。
- **Lunen** = **作者时**（事前）让你写策略；TrustLadder = **运行时**从你**真实审批行为**归纳策略（事后 / 揭示性偏好），且**可一键回退**。
- **通用审计日志** = 只写不读、供事后追责；TrustLadder 把审批流**回收**成「让明天少审」的资产，并加上「**反转即降级**」的负反馈闭环。
- 新原语是「**信任阶**」——没有任何被参考产品定义了它是什么、怎么挣、被辜负后怎么掉。
