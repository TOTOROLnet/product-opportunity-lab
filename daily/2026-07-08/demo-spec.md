# Demo 设计说明 — 2026-07-08

## 1. Demo 名称
ClaimLadder（追偿阶梯）

## 2. 一句话定位
把一团乱麻的消费纠纷，变成一份看得懂、可核验的「案卷」＋一条你能自己走完的「升级阶梯」——AI 帮你算清应得金额、给出法规依据、规划先礼后兵的谈判步骤，你逐级一键出手、全程留痕，AI 绝不替你越权自动执行。

## 3. 目标用户
遇到航班延误/取消、酒店超售、宽带长期故障、快递损坏、会员自动续费误扣等「理应拿回钱却被冗长客服流程劝退」的普通消费者。

## 4. 使用场景
用户在一次具体纠纷事件里（低频、事件驱动）打开 ClaimLadder：不知道自己能要多少、依据什么、下一步该怎么走、又不敢把个人信息交给黑箱 AI 全自动代办。ClaimLadder 在 3 分钟内给出「你能要 X 元 + 依据哪条 + 升级阶梯每一步说什么 + 每步留痕」。

## 5. Demo 策略分型（重要）

- 本机会属于：☑ 可视化/交互类产品　□ 抽象 / CLI / API / 基础设施类产品
- 采用的演示方式：**直接做可点击交互 Demo**。用户选场景/填几个关键字段 → 确定性地生成案卷（案件强度、透明算式、法规依据、升级阶梯）→ 逐级一键生成话术并标记进度 → 授权与留痕面板展示「透明 vs 黑箱」的差异。
- 演示的是**我们分析出的创新切入点**（透明案卷 + 升级阶梯 + 人在方向盘 + 可审计留痕），**不是** AirKaren（黑箱全自动代办）的克隆。

## 6. 核心流程
（用户从进入到"看懂价值"的主路径，3 分钟内可完成）
1. **选场景 + 填关键事实**：从 5 个预设场景（航班延误、酒店超售、宽带故障、快递损坏、会员误扣）中选一个，填 2–4 个关键字段（如延误时长、票价、订单金额）。也可点「用示例填充」一键体验。
2. **看案卷**：系统立即给出——①案件强度分（0–100，并列出加分/减分理由）②你应得金额的**透明算式**（逐项：依据哪条规则、怎么算）③法规/政策**引用卡**（mock 但贴近真实，显式标注「示例规则」）。
3. **走升级阶梯**：一条 3–4 级的 escalation ladder（普通客服 → 正式书面索赔 → 监管/仲裁 → 拒付/小额诉讼），每级显示「筹码计」增长；展开任一级可看到该渠道的话术/邮件草稿（可复制），点「标记为已发送 / 已回复」推进到下一级，并自动生成跟进提醒。
4. **授权与留痕**：底部「你做主」面板——展示每个动作会共享哪些个人数据（可逐项脱敏/关闭）、动作可逆性风险、以及一条随操作实时增长的审计日志（对照 AirKaren「黑箱代办」的 before/after 说明）。

## 7. 页面结构（<= 3 个主要视图，单页应用内切换）
- 视图 1 · **Intake（立案）**：场景选择卡片 + 关键字段表单 + 「用示例填充 / 生成案卷」按钮。
- 视图 2 · **Case & Ladder（案卷与阶梯）**：核心价值屏。左/上为案件强度 + 透明算式 + 法规引用；中部为升级阶梯（含筹码计与每级可展开话术）；此视图内含 Tab 或分区。
- 视图 3 · **Trust & Audit（授权与留痕）**：数据共享逐项授权 + 可逆性风险 + 实时审计日志 + 「透明 vs 黑箱」对比（作为视图 2 内的一个分区/抽屉呈现，逻辑上算第 3 个主视图）。

## 8. 关键交互
- 场景切换即时改变表单字段与后续案卷（确定性映射）。
- 「用示例填充」：一键写入合理默认值，降低体验门槛。
- 升级阶梯：点「标记已发送/已回复」逐级解锁，筹码计与审计日志实时更新；话术草稿一键复制。
- 数据脱敏开关：切换后审计日志与「计划共享数据」实时反映，直观展示「人在方向盘」。
- 全程**无任何自动发送/自动拨打**：所有动作都是用户点击触发，凸显与黑箱代办的本质差异。

## 9. 使用的 mock 数据
（说明数据来源与结构，全部为 mock，不接真实后端/外部 API）
- `src/data/scenarios.ts`：5 个场景，每个含：字段定义、案件强度算法输入、赔偿测算规则（逐项算式）、法规/政策引用卡（标注「示例规则，以实际法规为准」）、升级阶梯各级（渠道、对象、话术/邮件草稿模板、预计时效、筹码增量）。
- `src/logic/analyze.ts`：确定性「分析引擎」——把用户输入映射为案件强度分、金额算式、引用与阶梯。**在 UI 中明确标注**：真实产品此步由 LLM 完成分类/法规匹配/草稿生成，Demo 用规则模拟以保证可离线、可复现、不误导。
- 无任何真实法规数据库、无外部 API、无用户数据上传。

## 10. 明确不做什么
- 不做登录 / 用户系统
- 不接数据库 / 支付 / 外部私钥 / 真实 API / 真实法规库
- 不真的拨打电话、发送邮件或提交表单（刻意如此——这正是与 AirKaren 黑箱代办的本质区别）
- 不做生产级后端
- 不提供任何应被视为正式法律意见的内容（Demo 内含免责声明）

## 11. 成功体验标准
- 用户 3 分钟内能说出：「它帮我算清能要多少、依据什么、下一步先做什么再做什么，而且我全程掌控、看得见每一步」。
- 首屏即能感知「透明 + 你做主」与「黑箱全自动」的差异。
- 全流程无死链、无空壳页面；升级阶梯与授权留痕均可交互并实时反馈。
