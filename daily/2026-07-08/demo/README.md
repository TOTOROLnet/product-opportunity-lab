# ClaimLadder（追偿阶梯）— Demo

> 2026-07-08 · product-opportunity-lab 每日机会 Demo
> 一个「看得懂、你做主」的消费维权副驾：AI 帮你算清应得金额、给出法规依据、
> 规划先礼后兵的**升级阶梯**，你逐级一键出手、全程留痕——**不做黑箱全自动代办**。

## 这是什么

灵感来自 Product Hunt 雷达（2026-07-08）对 **AirKaren**（替你跟航司客服掰头维权的执行型代理）的观察。
本 Demo 的切入点与 AirKaren **相反**：不把打电话 / 发正式函件这些敏感动作交给黑箱 AI 全自动执行，
而是把执行型维权重构成——

1. **透明案卷**：案件强度分 + 你应得金额的逐项算式 + 法规 / 规则引用；
2. **升级阶梯**：一条先礼后兵、可视化「筹码」的谈判路径（客服 → 正式书面 → 监管 / 仲裁 → 拒付 / 诉讼），
   每一步都由你自己一键出手，ClaimLadder 只准备话术、算清依据；
3. **你做主 · 可审计留痕**：敏感数据逐项授权可脱敏，全程操作自动记入审计日志——
   这正是报告点名的「执行型消费代理头号风险（信任 / 授权 / 留痕）」，被做成首屏第一性体验。

覆盖 5 个跨行业场景：航班延误、酒店超售、宽带故障、快递损坏、会员自动续费误扣。

## 重要说明

- **纯前端静态站点**：无登录 / 数据库 / 支付 / 外部 API / 密钥。
- **不真的拨打电话 / 发邮件 / 提交表单**——这是刻意为之，也是与黑箱代办的本质区别。
- **所有规则、金额、法规引用均为演示用示例**，不构成法律意见。真实产品中这些步骤由 LLM
  基于可维护的规则库生成；此处用离线确定性规则模拟，保证可复现、不误导。

## 本地运行

```bash
npm install     # 安装依赖
npm run dev     # 本地开发预览（默认 http://localhost:5173）
npm run build   # 生产构建，产物在 dist/
npm run preview # 预览构建产物
```

## 技术栈

- Vite + React + TypeScript
- `vite.config.ts` 设 `base: './'`，保证部署到任意 GitHub Pages 子目录都能正确加载资源。

## 目录结构

```
src/
  main.tsx                  入口
  App.tsx                   状态机（立案 → 案卷）与整体编排
  types.ts                  领域类型
  data/scenarios.ts         5 个场景的字段定义与展示元数据（mock）
  logic/analyze.ts          确定性「分析引擎」：强度 / 金额 / 法规 / 阶梯（mock 规则）
  logic/format.ts           金额格式化
  components/
    Intake.tsx              立案：选场景 + 填关键事实
    StrengthMeter.tsx       案件强度可视化
    CompensationBreakdown.tsx  透明金额算式
    RegulationCards.tsx     法规 / 规则引用卡
    EscalationLadder.tsx    升级阶梯 + 筹码 + 话术 + 人工确认
    TrustAudit.tsx          授权脱敏 + 审计留痕 + 透明 vs 黑箱对比
```

## 3 分钟体验路径

1. 选一个场景（如「航班延误」）→ 点「用示例填充」→「生成我的案卷」。
2. 看案卷：案件强度、应得金额算式、法规依据。
3. 展开升级阶梯，复制某一级话术（你自己发送），点「标记完成」逐级解锁、看筹码增长。
4. 在底部「你做主」面板把某个敏感字段一键脱敏，观察审计日志实时记录。
