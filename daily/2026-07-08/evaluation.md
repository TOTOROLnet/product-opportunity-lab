## 7. Demo 的最大问题
规则与金额均为**示例**（EU261 等做了简化），真实产品的胜负手在于「准确、可维护、可更新的法规/规则库 + 靠谱金额测算」，这在 Demo 中被确定性 mock 替代（已在 UI 与 README 显著标注）。因此 Demo 验证的是**产品形态与价值主张**，而非法规准确性。

## 8. 是否建议用户人工体验
**建议。** 全流程 3 分钟内可跑通，最能体现价值的是「升级阶梯逐级解锁 + 筹码增长」与「脱敏 → 审计日志联动」两处交互，静态截图不足以传达，值得亲自点一遍。

## 9. 下一步建议
- 引入可维护、可溯源的真实规则库（分辖区 / 分行业），并让 LLM 做「分类 → 法规匹配 → 金额测算 → 草稿」的可解释生成。

- 探索中立可持续的变现（订阅 / 增值）而非抽佣，避免夸大金额、损害中立性。
