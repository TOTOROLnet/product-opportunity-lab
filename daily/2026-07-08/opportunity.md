## 2. 候选机会（至少 3 个，均为创新切入点而非照抄）

### 候选 A：ClaimLadder（追偿阶梯）— 会算账、有依据、你做主的「透明案卷 + 升级阶梯」维权副驾

- 创新切入点：把执行型维权的重心从「AI 全自动替你办（黑箱）」**反转为「AI 帮你建一份看得懂、可核验的案卷 + 一条清晰的升级阶梯，你逐级一键出手，全程留痕」**。核心可视化对象是三样东西：①**案件强度分**（你这案子有多硬）②**你应得金额的透明算式**（依据哪条规则、怎么算出来的）③**升级阶梯（escalation ladder）+ 筹码计（leverage meter）**（先礼后兵：普通客服→正式书面索赔→行业监管/仲裁→拒付/小额诉讼，每上一级你的筹码怎么变大）。
- 目标用户 / 场景：遇到航班延误、酒店超售、宽带长期故障、快递损坏、会员自动续费误扣等「理应拿回钱却被流程劝退」的普通消费者。
- 五维评分：痛点 5/5，机制新意 4/5，AI 核心度 4/5，一天可行性 5/5，启发价值 4/5 → 合计 **22/25**

### 候选 B：TrustGate（授权闸门）— 面向所有「执行型消费代理」的通用人工确认 + 可审计留痕层

- 创新切入点：不做某个维权垂类，而是做一层横向「代理动作确认与留痕面板」——任何执行型 AI 在「打电话/发正式函件/共享你的身份与订单信息」之前，先在 TrustGate 弹出「计划预览 + 逐字段数据脱敏授权 + 该动作可逆性风险 + 事后回执留痕」。
- 目标用户 / 场景：AirKaren 这类执行型代理的用户、以及做这类代理的团队（偏 B2B/基础设施）。
- 五维评分：痛点 4/5，机制新意 4/5，AI 核心度 3/5，一天可行性 4/5，启发价值 4/5 → 合计 **19/25**
- 备注：与 07-07 的「Reverso（Agent 动作可逆性/回滚）」主题重叠度偏高，AI 更偏「外挂闸门」而非核心，扣一档。

### 候选 C：Refund Radar（回款雷达）— 主动发现「你都不知道自己被欠了钱」的潜在索赔

- 创新切入点：AirKaren 是**被动**的（你得先知道自己有得赔、还得主动发起）；Refund Radar 反过来做**主动侦测**——（模拟）扫描你的订单/账单/邮件里的延误、降价、故障、自动续费误扣，主动冒出「这里你大概率能要回 ¥X」的机会卡，再交给索赔流程。
- 目标用户 / 场景：懒得逐笔盯账单、经常「多花了钱也不知道」的普通消费者。
- 五维评分：痛点 4/5，机制新意 4/5，AI 核心度 4/5，一天可行性 3/5，启发价值 4/5 → 合计 **19/25**
- 备注：「侦测」是最大亮点也是最大 Demo 难点——纯 mock 很难让「主动发现」显得可信，价值密度不如 A 直接。

### 候选 D：CS Judo（客服话术反制陪练）— 实时识别客服拖延套路并给你反制话术

- 创新切入点：把「和客服掰头」做成一个陪练/实时副驾：识别对方的拖延/踢皮球套路（「系统升级」「需要主管审批」），实时提示你的反制话术与该援引的规则。
- 五维评分：痛点 3/5，机制新意 3/5，AI 核心度 3/5，一天可行性 4/5，启发价值 3/5 → 合计 **16/25**

## 3. 最终选择

- 选中：**候选 A — ClaimLadder（追偿阶梯）**，总分 **22/25**
- 为什么选它（相对其他候选的优势）：
  1. **直接命中报告点出的头号风险**（信任/授权/留痕），并把它从「要补的漏洞」升维成「产品第一性卖点」——这是最有说服力、最反差于 AirKaren 的切入点。
  2. **痛点与价值密度最高**：单次事件就能量化交付「把 X 元要回来」，且用户全程看得懂，低频场景也能占住心智。
  3. **一天可行性最好**：核心是「结构化案卷 + 升级阶梯 + 透明算式」的可视化，纯前端 + mock 规则即可讲清价值，无需真拨打电话/接外部 API。
  4. B（偏 B2B、与 07-07 重叠）、C（Demo 可信度难）、D（价值偏薄）各有短板。

## 4. 深入论证（针对选中机会）

1. 目标用户：被「理应拿回钱、却被冗长客服流程劝退」的普通消费者。第一批用高频且金额可观的场景切入：航班延误/取消（欧盟 EU261、各国民航规则）、酒店超售、宽带/流媒体长期故障、会员自动续费误扣、快递损坏。
2. 核心痛点（真实、高频、可付费）：
   - **看不懂规则**：EU261、消保法、平台规则条款晦涩，不知道自己到底能要多少。
   - **不知道怎么开口、下一步怎么走**：第一封邮件被打发后就卡住，不知道何时该升级、升到哪。
   - **怕被坑、怕越权**：既怕自己说错话失去筹码，也不敢把身份/订单信息一股脑交给黑箱 AI 去「全自动代办」。
3. 现有产品为什么没解决好：
   - AirKaren / DoNotPay 式「全自动黑箱代办」把最敏感的动作（代打电话、代发正式法律函件、代填个人信息）交给用户看不见的 AI，**信任成本极高**，且抽佣模式有夸大金额的动机。
   - AirHelp/FlyClaim 式传统代维权只做航空、且抽走高额佣金，**用户既不透明也不掌控**。
   - 通用 ChatGPT 能写投诉信，但**不算金额、不给法规依据、不给升级路径、不留痕**——用户拿到一段模板后仍然卡在「然后呢」。
4. 为什么适合做轻量 Demo：核心价值是「**把一团乱麻 → 一份看得懂、可核验、有下一步的案卷**」的**可视化**，这天然适合纯前端交互演示：输入遭遇 → 生成案件强度/透明算式/法规依据/升级阶梯 → 逐级一键生成话术并标记进度 → 授权与留痕面板。全部可用 mock 规则库确定性地跑出来，无需任何外部 API/密钥。

## 5. 不照抄声明（四项必填）

1. **创新切入点是什么**：把执行型维权的产品重心，从「AI 全自动替你办（黑箱、不可见、airline-only）」**反转为「AI 帮你建透明可核验的案卷 + 升级阶梯，你逐级一键出手、全程留痕、人始终在方向盘上」**；并把「授权边界 + 可审计留痕 + 人工确认」做成**首屏第一性体验**而非事后补丁。
2. **解决的具体问题 / 场景**：普通消费者在维权时「看不懂规则、算不出应得金额、不知道下一步/何时升级、又怕把个人信息交给黑箱 AI」——ClaimLadder 一次性给出「你能要多少 + 依据哪条 + 先做什么再做什么 + 每步说什么 + 每步都留痕」。
3. **相对被参考产品（AirKaren）的增量价值（差异点）**：
   - **透明 vs 黑箱**：应得金额有可核验算式、有法规引用，用户看得懂而非「相信 AI」。
   - **人在方向盘 vs 全自动**：每个动作用户一键触发、可编辑、可跳过，AI 不越权自动拨打/发送。
   - **升级阶梯 + 筹码计**：把「先礼后兵」的谈判策略产品化、可视化（AirKaren 只描述「会追进度」，未把策略显性化）。
   - **跨行业通用 vs 航空绑定**：同一套「结构化案卷」能力覆盖宽带/酒店/电商/会员扣费。
   - **中立可持续变现**：透明案卷 + 订阅/增值，避免抽佣激励下夸大金额、损害中立性的死结。
4. **为什么这不是照抄 AirKaren**：AirKaren 的本质是「你把事全交给 AI，AI 在后台替你打电话/发函（你看不见过程）」；ClaimLadder 的本质是「AI 把你的案子摊在桌面上让你看懂、并给你一条能自己走完的升级路径，AI 全程不替你越权执行敏感动作」。两者在**信任模型、控制权归属、可见性、场景边界、变现激励**上都是相反的。若产出变成「一键全自动代打电话」，即视为照抄失败，必须重做。

## 6. 不确定性与风险

- 需求侧风险：维权低频、事件驱动，获客成本可能高；「看得懂的案卷」是否足以让用户放弃「全自动省事」的诱惑，需要验证。
- 落地 / 技术风险：真实产品需要**准确、可更新的法规/规则库**与靠谱的金额测算，错误引用法规会直接损害信任；跨行业规则维护成本高。Demo 用 mock 规则并**显式标注「示例规则，以实际法规为准」**规避误导。
- 我可能判断错的地方：也许相当一部分用户确实「只想甩手不想看懂」，那么黑箱全自动（AirKaren 方向）反而更契合他们；ClaimLadder 更适合「金额较大、愿意花 10 分钟掌控过程」的人群。真实市场可能需要「透明案卷」与「一键代办」并存，我先押注前者作为信任地基。
