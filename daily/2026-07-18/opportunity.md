## 2. 候选机会（至少 3 个，均为创新切入点而非照抄）

### 候选 A：明镜 GlassTutor — 可被学习者校正的「玻璃盒」自适应导师
- 创新切入点：不做「更聪明的黑盒自适应」，而是把**「AI 对你的认知模型」本身摊开成产品第一屏**——每个知识点的掌握度、置信度、以及「凭哪几次作答/证据得出这个判断」都可见；每道题都回答「**为什么现在考它**」；并给学习者一个**校正模型**的一等动作：「我其实会这个（刚才手滑）」「我想先攻这个分支」，一按即让学习路径**确定性地重新规划**。核心新原语：*自适应不再是你被动信任的黑盒，而是你和 AI 共同持有、可审计、可校正的一张认知地图。*
- 目标用户 / 场景：备考/自学一门有明确前置结构技能（编程、语言、标化考试）的自学者，苦于「系统凭一道错题误判我、反复操练已会内容、却看不见也改不了」。
- 五维评分：痛点 **4**/5（真实、高频、教育市场付费成熟，但属"软痛"），机制新意 **5**/5（玻璃盒 + 可校正认知模型是全新品类，Paradigm/多邻国等都把自适应锁死在黑盒），AI 核心度 **5**/5（自适应认知建模就是 AI 核心，AI-native 2C），一天可行性 **5**/5（可交互技能图谱 + mock 题 + **真实确定性掌握度引擎**，纯前端可跑），启发价值 **4**/5（"把 AI 对用户的模型做成玻璃盒且可校正"可迁移到任意个性化/推荐/agent 产品）→ 合计 **23/25**

### 候选 B：值守面板 AgentDeck — 按「爆炸半径」分级摩擦的并行 agent 审批舱
- 创新切入点：针对报告点名的「一键批准易误授权」，做一个软件版并行 agent 审批舱，核心不是 RGB 状态灯（那是 Codex Micro 本体），而是**按可逆性与爆炸半径给每个待批动作分级施加摩擦**——本地可逆编辑可批量清空/策略自动放行，触及密钥/生产/不可逆外部副作用（发信/花钱/迁移）的动作**强制展开、加阻力**，让你无法把危险动作当成前面 50 个无害动作一样橡皮图章。
- 目标用户 / 场景：同时挂多个 coding agent 的高级开发者/团队。
- 五维评分：痛点 **4**/5，机制新意 **3**/5（分级摩擦是好切口，但 agent 监督/授权是本实验室重复区，平台内建竞争激烈），AI 核心度 **4**/5，一天可行性 **5**/5，启发价值 **3**/5 → 合计 **19/25**
- （否决理由：与本实验室 07-12 明约、07-14 值当、07-08 ClaimLadder 的「人在回路 + 授权/对账」失败模式高度同质，增量有限；且护城河被平台方内建挤压。）

### 候选 C：缓存工坊 CacheCraft — Kimi K3 cache-hit 成本可视化优化器
- 创新切入点：针对 K3「cache-hit \$0.30 / miss \$3.00」10x 差价，做一个把提示词/上下文按「稳定 vs 易变」重排、可视化命中率与省钱幅度的工具。

## 3. 最终选择

- 选中：**候选 A — 明镜 GlassTutor（可被学习者校正的玻璃盒自适应导师）**，总分 **23/25**
- 为什么选它（相对其他候选的优势）：
  1. **踩今日唯一达标的 2C 信号，且攻其要害**：Paradigm 是今天唯一达标的 2C 新品，报告自己把「自适应效果需长期验证」列为其头号风险——A 不绕开这个风险，而是把它当作产品切口正面解决（透明 + 可校正）。
  2. **打破本实验室系统性偏科**：近 10 天产出几乎全在「agent 监督/授权/记忆治理/对账」（07-08~07-17 大量同质），`lab-focus.md` 明确要求「不要只从技术向选题而系统性忽略 2C」。B/D 会再度落回重复区，C 靠臆造成本数字。A 是唯一既新鲜、又高分、又能诚实演示的方向。
  3. **可诚实地可视化（关键）**：A 可实现一个**真实的确定性掌握度引擎**（作答→贝叶斯式更新→重新选题），before/after 的「省下多少无效操练」是**算出来的、不是编的**——延续本实验室最佳产出（07-17 常青、07-13 记衡的真引擎）的诚实标准。
  4. **机制新意与可迁移性最高**：「把 AI 对用户的内部模型做成玻璃盒 + 可校正」是一条能迁移到推荐、个性化、agent memory 的通用原语，启发价值超出教育本身。

## 4. 深入论证（针对选中机会）

1. **目标用户**：正在自学一门**有清晰前置依赖结构**的技能的人——备考（SAT/GRE 数学）、学编程（如 SQL、算法）、学外语语法。他们已经受够了「僵化课表」，转向自适应产品，却又困在新的黑盒里。
2. **核心痛点（真实、高频、可付费）**：
   - **误判无法申诉**：手滑/看错题被判「不会」，系统据此反复喂已掌握内容，浪费我最稀缺的东西——时间与耐心；我看不见它凭什么这么判，也没法纠正。
   - **信任崩塌 = 留存崩塌**：报告点名的「留存难」根因之一，就是学习者对「它到底懂不懂我」失去信心，于是流失。透明与掌控感是留存的前提。
   - 教育/备考是被验证的付费市场（Paradigm 靠免费层 + haggle 获客，说明付费意愿真实存在）。
3. **现有产品为什么没解决好**：多邻国、Paradigm 乃至各类自适应引擎，都把「对你的能力估计」**锁在服务端黑盒**里——对用户只暴露「下一题」这个结果，不暴露「我对你的判断 + 为什么」，更不给「你来改」的入口。这是行业默认设计，不是疏忽——但它恰恰制造了不可申诉的误判与不可解释的路径。
4. **为什么适合做轻量 Demo**：认知模型 + 选题逻辑可用**确定性算法**在纯前端完整实现（IRT-lite 掌握度更新 + 前置依赖 + 不确定性驱动选题）；技能图谱、题库、证据日志全部 mock；「校正模型 → 路径重规划」「黑盒 vs 玻璃盒 before/after」都能被真实计算并可视化，无需任何后端/LLM/外部 API。

## 5. 不照抄声明（四项必填）

1. **创新切入点是什么**：把「AI 对学习者的认知模型」从服务端黑盒**翻转成产品第一屏的玻璃盒**，并赋予学习者**一等的校正权**（改掌握度、改学习意图、标记"手滑"），使自适应从「你被动接受的结果」变成「你和 AI 共同持有、可审计、可校正的一张地图」。
2. **解决的具体问题 / 场景**：自学者被自适应系统**不可见、不可申诉地误判**，导致时间浪费与信任/留存崩塌——明镜让每一次判断可见、可解释、可当场纠正，并把纠正确定性地反映到后续学习路径。
3. **相对被参考产品（Paradigm / 通用自适应引擎）的增量价值**：
   - Paradigm 卖「实时重写课程 + 真实 VPS 沙箱 + haggle 定价」，其自适应**对用户不透明**；明镜卖「透明 + 可校正的认知模型」，是一层**薄的、可套在任意学习内容上的适应层**，主张「你能看见并修改 AI 对你的判断」。
   - 差异点：Paradigm 优化「生成更多/更贴合的内容」；明镜优化「让 AI 对你的估计正确且可控」——攻的是留存根因（信任），不是内容量。
4. **为什么这不是照抄 Paradigm（本质区别）**：Paradigm 是一个**全栈自适应学习平台**（内容生成 + 工具沙箱 + 定价玩法），自适应是黑盒；明镜**不生成课程、不开 VPS、不玩定价**，它是一个**透明可校正的适应/建模层**，产品的全部主张就是「把黑盒变玻璃盒并交出校正权」——这是 Paradigm 明确没有做、且与其重内容/重工具路线正交的方向。若把明镜的透明层去掉，它就退化为普通自适应，正说明「透明 + 可校正」才是它的本体，而非对 Paradigm 的复刻。

## 6. 不确定性与风险

- **需求侧风险**：多数学习者可能「只想要结果、懒得看/改模型」——玻璃盒可能是进阶用户才珍视的功能。缓解：默认极简（只在误判发生时轻量提示"这判断对吗？"），把透明做成"需要时可展开"而非负担。
- **落地 / 技术风险**：真实产品里，掌握度估计要接真实作答与内容生成；Demo 用的是确定性 mock 引擎，能讲清机制但不能证明真实自适应质量（这点会在 evaluation 诚实标注）。
- **我可能判断错的地方**：也许「透明度」对留存的贡献没我想的大，真正决定留存的是内容质量与情感陪伴（Clover 那类导师人格）。若如此，玻璃盒会退化为差异化点而非核心杠杆。
- **客观性提醒**：本机会的价值论证部分建立在「报告事实：Paradigm 自适应需长期验证 + 教育留存难」之上，但「透明可校正能显著改善留存」是**我的判断/假设**，非报告结论，需真实用户验证。

- 是否达到 16/25 门槛：**是**
