# Demo 设计说明 — 2026-07-18

## 1. Demo 名称
**明镜 GlassTutor** — 可被学习者校正的「玻璃盒」自适应导师。

## 2. 一句话定位
别的自适应学习把「它以为你会什么」锁进服务端黑盒；明镜把这份「AI 对你的认知模型」摊开成第一屏、告诉你每道题「为什么现在考它」，并让你亲手校正它对你的判断。

## 3. 目标用户
自学一门**有清晰前置依赖结构**技能（编程、语言语法、标化考试数学）的自学者；受够了黑盒自适应「凭一道手滑错题误判我、反复操练已会内容、我却看不见也改不了」。

## 4. 使用场景
我正在学 SQL 数据查询（从 SELECT / WHERE 到 JOIN / GROUP BY / 子查询 / 窗口函数）。我做几道题，想知道：系统现在到底认为我会什么、下一题为什么考它；当它误判我（手滑）时，我要能当场纠正，并看到学习路径随之改变。

## 5. Demo 策略分型（重要）

- 本机会属于：**■ 可视化 / 交互类产品**（同时含 before/after 价值可视化）
- 采用的演示方式：
  - 主体是**可点击的交互 Demo**：真实作答 → 认知模型确定性更新 → 可视化 → 重新选题。
  - 叠加一屏 **before/after 对比**（黑盒自适应 vs 明镜玻璃盒），用**真实引擎算出的「无效操练题数 / 省下时间」**做价值可视化，不臆造。
- 演示的是**我们分析出的创新切入点**（透明 + 可校正的认知模型），不是 Paradigm 的克隆（不生成课程、不开 VPS、不玩定价）。

## 6. 核心流程
（用户从进入到「看懂价值」的主路径，3 分钟内可完成）
1. **进入「学习舱」**：中间是一道 SQL mock 题，左侧是我的技能图谱（每个节点按掌握度着色，附置信度）。右侧「为什么现在考它」面板展示引擎的选题理由（低掌握 / 前置已满足 / 不确定性高 / 我声明的兴趣）。
2. **作答 2–3 题**：每次作答后，被影响的知识点掌握度**实时更新**，并在证据日志里留痕（「第 3 题答对 → JOIN 掌握度 0.42→0.58」）。看到「AI 对我的判断在变，且我知道它凭什么变」。
3. **触发误判并校正**：故意/被引导答错一道其实会的题 → 掌握度被下调 → 点「我其实会这个（刚才手滑）」→ 引擎**确定性地**回调该点掌握度并重排后续题目；或点「我想先攻窗口函数」→ 选题权重改变、路径重规划。
4. **切到「认知地图」**：看 AI 对我的**完整模型**——每个技能的掌握度、置信度、证据日志、是否过期（衰减），且每项可手动校正。这就是「玻璃盒」。
5. **切到「对比」**：同一串作答序列跑「黑盒（不能看/不能改）」vs「明镜（一次手滑校正）」，看引擎算出黑盒多喂了几道**无效操练题**、明镜省下多少。

## 7. 页面结构（<= 3 个主要页面 / Tab）
- **页面 1 · 学习舱（Study）**：题卡 + 技能图谱缩略 + 「为什么现在考它」+ 作答后更新动画 + 校正入口。核心交互主场。
- **页面 2 · 认知地图（Knowledge Map，玻璃盒）**：AI 对学习者的完整模型；每个技能可展开证据日志、可手动校正掌握度/兴趣；显示前置依赖连线与掌握度着色。
- **页面 3 · 对比（Black-box vs GlassTutor）**：before/after，真实引擎计算的无效操练题数与节省，配一句"透明+可校正到底省了什么"。

## 8. 关键交互
- **作答**：单选题；答后即时更新受影响技能掌握度（确定性 IRT-lite 更新），并写证据日志。
- **「为什么现在考它」**：展示候选题的选题打分（掌握度缺口 × 前置就绪 × 不确定性 × 兴趣权重），可见即可解释。
- **校正模型（一等动作）**：
  - 「我其实会这个（手滑）」→ 回调该技能掌握度 + 降低其被重复抽中的权重。
  - 「我想先学 X」→ 提升 X 及其前置的兴趣权重，路径重规划。
  - 认知地图里可直接拖动/设定某技能掌握度（模拟"我告诉 AI 真实水平"）。
- **对比重放**：一键把「黑盒/玻璃盒」两种策略在同一作答序列上跑完，输出可比较的量化结果。

## 9. 使用的 mock 数据
（全部为 mock，不接真实后端 / LLM / 外部 API）
- `data/skills.ts`：SQL 技能图谱（6–8 个节点：SELECT、WHERE、ORDER BY、聚合、GROUP BY、JOIN、子查询、窗口函数）+ 前置依赖边 + 每节点初始掌握度/置信度。
- `data/questions.ts`：每个技能挂 2–3 道 mock 单选题（题干、选项、正确项、难度、所测技能）。
- `logic/engine.ts`：**确定性掌握度引擎**——
  - 掌握度更新：`p' = clamp(p + lr·(outcome - p_expected))`，`p_expected` 由掌握度与题目难度经 logistic 得出（IRT-lite）；置信度随观测数上升。
  - 衰减：距上次练习越久掌握度轻微回落（过期标记）。
  - 选题：`score = wGap·(1-p) + wUncertain·(1-confidence) + wPrereq·prereqReady + wInterest·interest`，取最高者；面板展示各项分值。
  - 校正：手滑 = 撤销上一次下调 + 降权；改兴趣 = 调 interest 权重；均触发确定性重排。
  - before/after：同一 answer trace 下，统计"对已掌握技能（p≥0.8）仍被抽中的题数"作为**无效操练题数**，黑盒不校正 vs 玻璃盒一次校正的差值即节省。

## 10. 明确不做什么
- 不做登录 / 用户系统。
- 不接数据库 / 支付 / 外部私钥 / 真实 API / LLM。
- 不生成真实课程内容、不开云 VPS（那是 Paradigm 的重资产，不是本机会主张）。
- 不做「和 AI 讨价还价」定价玩法。
- 不做生产级后端；全部数据 mock、逻辑在前端确定性运行。

## 11. 成功体验标准
- 用户 3 分钟内能说出：**解决什么**（黑盒自适应误判不可见/不可申诉）、**怎么用**（作答→看 AI 对我的判断+理由→当场校正）、**增量在哪**（对比屏里"玻璃盒省下的无效操练题数"是真算出来的）。
- 首屏能一眼看出这是「透明 + 可校正的自适应」，而不是又一个套壳聊天式学习工具。
- 「校正模型」是可点、且能立刻看到路径变化的一等动作，而非说明文字。
