# 明镜 GlassTutor — 可被你校正的「玻璃盒」自适应导师（静态 Demo）

> 2026-07-18 每日产品机会循环产出。信号来源：Product Hunt AI 雷达当日 2C 达标品 **Paradigm**（自适应学习）。
> 本 Demo 不是 Paradigm 的克隆——它只做一层**透明 + 可校正的适应/建模层**（详见同目录 `../opportunity.md`）。

## 这是什么

别的自适应学习产品把「AI 以为你会什么」锁在服务端黑盒里，对你只吐出「下一题」。
一旦它凭一道**手滑错题**误判你，就会反复操练你早已掌握的内容，浪费你的时间——而你**看不见、也改不了**。

**明镜** 把这份「AI 对你的认知模型」翻转成产品第一屏：

- **透明**：每个知识点的掌握度、置信度、以及「凭哪几次作答得出这个判断」的证据日志都可见；每道题都回答「**为什么现在考它**」。
- **可校正**：你有一等的校正权——「我其实会这个（刚才手滑）」「我想先攻窗口函数」「我在这个技能上其实是 X 水平」，一按即让学习路径**确定性地重新规划**。

## 三个页面（3 分钟看懂）

1. **学习舱**：答 SQL 题；每次作答后看 AI 对你的判断如何更新、下一题为什么被选中；答错时可当场标记「手滑」并看到右侧「为什么现在考它」立刻变化。
2. **认知地图（玻璃盒）**：AI 对你的完整模型——技能图谱 + 每个技能的掌握度/置信度/证据日志，且可手动校正（滑块设定真实水平、调整学习意图）。
3. **黑盒 vs 玻璃盒**：同一个确定性引擎、同一串作答、唯一区别是「能否当场校正手滑」，看**引擎算出**的「无效操练题数」差多少。

## 诚实说明（不是假逻辑）

`src/logic/engine.ts` 是一个**确定性掌握度引擎**：

- 掌握度更新用 IRT-lite（logistic 期望 + 学习率随置信度递减）；
- 选题分 = 缺口 × 不确定性 × 前置就绪度（门控）× 兴趣 × 近期降权；
- 校正会确定性地回调掌握度 / 调整权重并重排路径；
- 对比页的「省下 N 题」是引擎跑出来的（改题库或真实水平，结论会随之变化），不是写死的。

没有随机、没有后端、没有 LLM。任何人做同样操作都得到同样结果。

## 本地运行

```bash
npm install
npm run dev      # 本地开发预览（默认 http://localhost:5173）
npm run build    # 产出静态站点到 dist/
npm run preview  # 预览 build 产物
```

## 技术栈与约束

- Vite + React + TypeScript。
- `vite.config.ts` 设 `base: './'`，保证部署到任意 GitHub Pages 子目录都能正确加载资源。
- 纯前端静态站点：**不接**登录 / 数据库 / 支付 / 私钥 / 真实 API / LLM。全部数据 mock（SQL 技能图谱 + 题库）。

## 目录

```
src/
  main.tsx              入口
  App.tsx               会话状态 + Tab 导航
  types.ts              类型定义
  index.css             样式
  data/
    skills.ts           SQL 技能图谱（含前置依赖、对比用的隐藏"真实水平"）
    questions.ts        mock 题库
  logic/
    engine.ts           确定性掌握度引擎（更新 / 选题 / 校正 / 对比模拟）
  components/
    shared.tsx          掌握度色、掌握度条、SVG 技能图谱
    StudyView.tsx       学习舱
    KnowledgeMapView.tsx 认知地图（玻璃盒）
    CompareView.tsx     黑盒 vs 玻璃盒
```
