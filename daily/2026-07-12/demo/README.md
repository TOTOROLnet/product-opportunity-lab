# 明约 Mandate — 自主办公 agent 的「授权对账」透镜

> product-opportunity-lab · 2026-07-12 每日机会 Demo（纯前端静态原型，全部数据为 mock）

## 这是什么

**明约 Mandate** 是一个针对**自主办公 agent**（以 2026-07-12 radar 唯一高价值新品
OpenAI **ChatGPT Work** 为代表：批准计划 → 自主执行数小时 → 交付成品）的**执行后对账透镜**。

ChatGPT Work 的审批是**前向**的——你先批准一份计划、途中批准几个动作。但 agent 自主跑了
数小时后，**没有一个反向对账层**证明「我最终拿到的成品，仍落在我当初签字批准的计划边界内」。
**批准计划 ≠ 保证执行没有偏离。** 明约就补这层空档：

- 把你批准过的那份计划当成一份**有约束力的授权契约（mandate）**；
- agent 跑完后，把「实际执行」逐步对回「你批准的计划」，标注 **守约 / 微调 / 越界** 三态；
- 照出**沉默偏离**：范围扩张、未授权对外动作（真发了邮件而非拟稿）、擅自替换数据源等；
- 在**交付口 gating**：存在未处置的越界 → 交付暂停，你可对每个越界项**签署放行 / 驳回重做**，
  把「让资深判断留在回路里」做成可点的动作。

它**不执行任何办公任务、不产出成品**，只对既有的「计划 / 执行 / 成品」三件套做对账——
这也是它与 ChatGPT Work 的本质区别：一个是做事的 agent，一个是验证它没跑偏的裁判。

## 怎么体验（3 分钟）

1. 默认进入「出海营销 Brief」场景，左侧是 agent 交付的**成品**（看起来已完成、可直接发出）。
2. 用顶部 **裸交付 / 明约透镜** toggle 对比：裸交付只见光鲜成品；打开透镜后右侧出现
   「授权契约对账」，把批准计划每一步与实际执行并排，并在顶部给出「可否直接交付」裁定。
3. 点成品里任意一格（如「已发送预热邮件」）→ 反查它由哪一步计划产生、是否越界。
4. 对 🔴 越界项点「签署放行」或「驳回重做」，观察顶部裁定条实时变化（人在回路）。
5. 切到「季度费用报表」（未授权数据源）与「周会纪要（对照组）」——对照组完全守约，
   透镜会明确背书「可直接采用」，证明它**不是逢事必报的狼来了**。

## 本地运行

```bash
npm install
npm run dev      # 本地开发预览
npm run build    # 生产构建，产物在 dist/
npm run preview  # 预览构建产物
```

- 技术栈：Vite + React + TypeScript。
- `vite.config.ts` 设 `base: './'`（相对路径），保证部署到任意 GitHub Pages 子目录都能正确加载资源。

## 目录结构

```
src/
  data/scenarios.ts        # 3 个标注场景（2 个跑偏 + 1 个对照组），全部 mock
  logic/mandate.ts         # 对账摘要 / 交付裁定 / 标签逻辑（纯函数）
  components/
    ScenarioTabs.tsx       # 场景切换
    SummaryBar.tsx         # 守约/微调/越界计数 + 可否直接交付裁定
    DeliverablePanel.tsx   # agent 交付的成品（元素可点击反查）
    ReconciliationPanel.tsx# 授权契约对账（计划 vs 执行 + 签署/驳回）
  types.ts                 # 类型定义
  App.tsx                  # 状态与布局
```

## 边界（明确不做）

- 不做登录 / 用户系统，不接数据库 / 支付 / 外部私钥 / 真实 API / 真实 agent 运行时。
- 不真正执行办公任务、不产出真实成品；不臆造 token / 成本数字。
- 所有「计划 / 执行 / 成品」均为演示用 mock 数据。
