# 并笔 CoBaton — 人机共写的「执笔权」协调引擎（Demo）

> 垫在任意编辑器**之下**的冲突消解层。人和 AI agent 在同一份活文档上并行编辑时，保证你
> **永不被打断**、你的编辑**永不被静默丢弃**、agent 的活儿**永不白费**。
>
> 本 Demo 为 product-opportunity-lab 2026-07-19 每日机会产物，riding 报告信号
> 「人机共编面（OpenMarkdown）」与趋势「AI 从聊天框走向共享文档 / 画布」。

## 这是什么 / 解决什么

今天用 Cursor / Claude Code 等和 AI 共同改同一份文件时，常发生四类刺痛摩擦：

1. agent 一次重写整段，**你正在写的内容被静默覆盖**（编辑丢失）；
2. agent 大改导致**你的光标/心流被打断**；
3. agent 埋头重写一段，而你几秒后就把它删了（**白做**）；
4. 你和 agent 谁该让路、延后的改写要不要基于你的最新改动重算——**没人做这层协调**。

OpenMarkdown 之类共编产品用「按小节隔离 + 不闪烁渲染」**回避**冲突，但没有**消解**冲突。
CoBaton 不是编辑器，而是编辑器之下缺失的**协调协议**：区域所有权状态机 + 接力棒移交 +
延后 / 再校验。它保证四条：不动你活跃区 / 不丢你的编辑 / 不浪费在你要改的区 / 延后编辑必对齐你的最新改动。

## 演示方式（纯前端「模拟体验 + 价值可视化」）

同一条脚本化的「人机共写事件序列」，分别喂给两种模式引擎，差异**由引擎算出、非硬编码**：

- **① 共写台**：分小节的《退款 SOP》活文档 + 模式开关【朴素 ↔ 并笔】+ 时间轴回放；每步浮出「刚发生了什么」，实时着色人活跃区 / agent 落改 / 延后 / 丢失。
- **② 执笔权地图（玻璃盒）**：把每个小节的所有权、延后队列（接力棒）、逐事件日志摊开可见。
- **③ 对比（Before / After）**：两模式的可测指标并排——丢失编辑 / 被打断 / 白做 / 保留率 / 一致性。

核心引擎在 `src/logic/engine.ts`（确定性归约），数据在 `src/data/`。

## 本地运行

```bash
npm install
npm run dev      # 本地开发预览
npm run build    # 产出 dist/（vite.config.ts 已设 base:'./'，可部署到任意子目录）
npm run preview  # 预览构建产物
```

## 明确不做

- 不做真正的富文本编辑器 / CRDT / OT 生产实现；不接真实编辑器、真实 agent、真实 LLM。
- 不做登录 / 数据库 / 支付 / 外部私钥 / 后端；全部数据 mock。
- 本 Demo 是「执笔权协调协议」的**可视化模拟**，用于讲清创新切入点与增量价值，非生产系统。

## 技术栈

Vite + React + TypeScript（React 18）。`vite.config.ts` 设 `base: './'`。
