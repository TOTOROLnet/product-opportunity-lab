## 2. 候选机会（至少 3 个，均为创新切入点而非照抄）

### 候选 A：榫卯 Tenon —— 多 agent 共享工件的「接口契约 + 漂移哨兵」
- 创新切入点：在 Blaxel Agent Drive 这类**多 agent 共享盘**之上，架一层**生产者→消费者的工件接口契约**。
  系统为每个被共享的工件推断/登记一份 schema 契约（字段、类型、单位、取值域、状态=draft/final/superseded），
  消费者 agent 绑定契约来读；当生产者 agent 的输出**漂移**出契约（改名/改类型/改单位/删字段/新增枚举）时，
  哨兵在**写入时**（而非下游炸掉时）拦截，给出**下游影响图 + 最小迁移补丁**，阻止坏工件在 swarm 里扩散。
- 目标用户 / 场景：搭多 agent 流水线的开发者 / agent 平台团队（研究→分析→报告、抓取→清洗→建模等），
  多个 agent 通过共享盘交换 JSON/笔记/工具输出。
- 五维评分：痛点 4/5，机制新意 5/5，方向相关 4/5，一天可行性 5/5，启发价值 4/5 → 合计 **22/25**

### 候选 B：定义账本 Lexicon —— 把「团队纠错」沉淀成可复用的语义契约
- 创新切入点：受 PromptQL「纠错即沉淀」启发，但聚焦**团队级语义定义**：每次有人纠正 AI 对某个业务概念的
  理解（「活跃用户=28 天内登录，不是 30 天」），系统把它固化成一条**带归属、带作用域、可复用**的团队定义，
  下次任何人问到触及该定义的问题，AI 自动套用并显式标注「已应用团队知识 X（谁/何时/为何加）」，并检测**定义冲突**。
- 目标用户 / 场景：用共享 AI 做数据问答/分析的团队。
- 五维评分：痛点 4/5，机制新意 3/5，方向相关 4/5，一天可行性 4/5，启发价值 3/5 → 合计 **18/25**
- 减分原因：与本实验室 07-16 师承（纠错→规则）、07-17 常青（活文档）机制重叠，「语义层/指标层」市面已有近似物（dbt metrics 等），新意受限。

### 候选 C：合买台 GroupPick —— 让一群人的 AI 收敛到一个可辩护的共同购买决策（2C）
- 创新切入点：受 Basement「同页社交 + agent-to-agent」启发，但不做浏览器也不做代付，只切**群体决策收敛**：
  多人各自扔进候选与约束（预算/必须项/雷区），AI 抽取隐含偏好、生成共享决策矩阵、**主动指出大家分歧最大的取舍点**，
  并给出帕累托折中的推荐 + 理由，把「群聊里贴一堆链接、意见散乱」收敛成一个说得清的共同选择。
- 目标用户 / 场景：一起买大件/旅行/礼物的朋友或家庭。
- 五维评分：痛点 3/5，机制新意 3/5，方向相关 4/5，一天可行性 3/5，启发价值 3/5 → 合计 **16/25**
- 减分原因：群体购买为低频/情景性需求；「决策矩阵」易被视作花哨表格，AI 核心度需靠偏好抽取+冲突识别撑住，
  在一天 mock demo 里较难做得可信；且连续第三天做 2C，边际启发下降。

## 3. 最终选择

- 选中：**候选 A — 榫卯 Tenon**，总分 **22/25**
- 为什么选它（相对其他候选的优势）：
  1. **机制新意最高（5/5）**：把「多 agent 协作会坏在哪」从大家熟悉的「算力/隔离/授权/协调」挪到一个
     几乎无人认领的位置——**跨 agent 工件的接口契约与漂移**。它不是又一个治理面板（刻意避开今日最拥挤、
     且本实验室已过度使用的授权/审计家族），也不是共享盘本身的克隆。
  2. **可行性最高（5/5）**：漂移检测本质是确定性的 schema diff + 依赖图上的影响传播，可用纯前端
     「模拟体验 + before/after 价值可视化」诚实呈现，数字全部来自真实引擎而非口头声称。
  3. **方向正中 lab-focus**：多 agent 协作、context/tool-use 基础设施、给 AI 用的开发者工具——都是硬门槛内的核心方向。
  4. 相较 B（重叠已做机制）与 C（低频 + 第三天 2C），A 在新意、可行性、启发三项同时占优。

## 4. 深入论证（针对选中机会）

1. 目标用户：正在搭**多 agent 流水线/编排**的开发者与平台团队——多个 agent（或多个 sandbox）通过一块
   共享盘交换中间工件（findings.json、metrics.json、report.md、工具输出…）。
2. 核心痛点（真实、高频、可付费）：共享盘让「交接」变简单了，却让「**误读**」变隐蔽了。生产者 agent
   一旦悄悄改了某工件的**形状/单位/语义**（这在 agent 自主生成结构时极常发生），下游消费者 agent 不会报错，
   而是**继续跑、给出貌似合理却错误的结果**；错误在流水线里传播多步后才以「报告数字离谱」的形式暴露，
   排障要沿 swarm 反向追很多步——这是多 agent 系统里最贵、最难查的一类失败。
3. 现有产品为什么没解决好：
   - Blaxel Agent Drive / box / CreateOS 解决的是**存储与隔离**（把盘/算力做好），不管盘上工件彼此是否兼容；
   - valv / Kastra / HOL Guard 解决的是**动作授权**（agent 能否对外部资源做某动作），不管 agent 之间交换的**数据结构**；
   - RAG / 记忆层解决的是**检索**，不做**生产者↔消费者的接口约束**。
   → 「多 agent 工件的接口契约 + 写入时漂移拦截 + 下游影响分析」是一层**结构性缺失**，没有产品把它当主线。
4. 为什么适合做轻量 Demo：可用一个 3-agent 流水线（研究员→分析师→报告员）作 mock corpus，
   预置几种典型漂移（改名 `score→confidence`、单位 `%→0-1`、删字段、新增枚举），用一个确定性引擎算出
   「关/开 榫卯」两种世界的差异（静默错误数、检测延迟=从改动到暴露隔了几步、被保护的下游 agent 数、最小迁移补丁），
   before/after 一目了然，3 分钟讲清「解决什么 / 怎么用 / 增量在哪」。

## 5. 不照抄声明（四项必填）

1. **创新切入点是什么**：给多 agent 共享盘上的每个工件建立**生产者→消费者接口契约**，并用**写入时漂移哨兵**
   在坏工件扩散前拦截、给出下游影响图与最小迁移补丁。类比：它是多 agent 共享工作区的**类型系统 + 集成测试**。
2. **解决的具体问题 / 场景**：多 agent 通过共享盘交换工件时，生产者悄改工件形状/单位/语义 → 下游静默误读 →
   错误跨多步放大、排障极难。榫卯把这类**静默运行时腐坏**变成**写入时可捕获、可解释、可迁移**的事件。
3. **相对被参考产品的增量价值（差异点）**：
   - vs **Blaxel Agent Drive**：它给「能并发读写的管道」；榫卯给「管道里工件彼此契合的保证」——正交且互补，不替代。
   - vs **valv / Kastra / HOL Guard（治理栈）**：它们授权/审计**动作**；榫卯校验 agent 间**数据结构契约**——
     无审批、无策略、无权限审计，是「协作正确性」而非「权限治理」。
   - vs **RAG/记忆层**：它们做检索；榫卯做接口约束与漂移。
4. **为什么这不是照抄 Blaxel（本质区别）**：Blaxel 是存储/隔离基础设施（挂载、并发、扩容）；
   榫卯是正确性/接口层（契约推断、漂移检测、影响分析、迁移建议）。二者关注点相反、可叠加使用。
   同时我**刻意没有**做今天最拥挤、且本实验室已过度使用的治理面板（valv/Kastra/HOL Guard 家族），
   以避免克隆与家族过载。

## 6. 不确定性与风险

- 需求侧风险：**多 agent 共享盘仍偏早期**（Blaxel 自身处于私有预览），此机会带一定前瞻性；
  当下痛点可能集中在少数重度多 agent 团队，规模化时点存疑。缓解：先服务这批高痛用户，随生态成熟放量。
- 落地 / 技术风险：真实环境里「契约」既可**显式声明**也需**自动推断**；agent 输出天然半结构化，
  自动推断 schema 会有噪声与误判，需保留「这是建议、可忽略/可覆写」的余地；语义级漂移（同名不同义）比
  结构级漂移更难自动判定，真实产品可能需 LLM 辅助推断意图，存在误报。
- 我可能判断错的地方：
  - 也许生态会走向「统一工件 schema 标准」使问题在协议层消解，中间层价值被挤压（类似 MCP 之于工具权限）；
  - 也许多数团队会用「单一编排框架强约束工件格式」规避此问题，使独立产品的市场变窄；
  - 也许写入时拦截过严会拖慢 agent 迭代速度，需要「软告警 vs 硬阻断」的细致分级。

## 7. 门槛结论

- 最高分：22/25（候选 A 榫卯 Tenon）
- 是否达到 16/25 门槛：**是**
- 结论：**进入 Demo 开发**（Loop 2–5）。演示策略＝抽象/基础设施类 → 纯前端「模拟体验 + 价值可视化」
  （共享盘视图 + 漂移哨兵 before/after + 契约台账），演示的是我们的创新切入点（接口契约/漂移），
  不是 Blaxel 存储层的克隆，也不是治理面板。
