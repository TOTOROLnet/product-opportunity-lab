# 2C 消费应用监测目标

本文件**仅**供 2C 独立 Automation 使用。与 `config/focus.md`（B2B 主轨）分属两次 Agent 运行，**不得在同一次任务中同时加载**。

目标：从 Product Hunt 每日新品中，筛出对**消费级 AI 产品创业**有启发的方向——用户是谁、痛点是否真实、能否形成留存与付费，而不是对 API / Agent 平台的机制迁移价值。

---

## 重点关注（2C）

### 1. 个人 AI 助理（消费向）

* 面向个人（非企业 IT）的任务代理、生活助理、跨 App 自动化
* 语音 / 移动 / 桌面消费入口
* 个人记忆、偏好、 proactive 提醒（消费场景）

判断：用户是否愿意把日常决策交给它？高频场景是什么？

### 2. 教育 / 学习

* AI 导师、自适应学习、练习与反馈闭环
* 语言、技能、考试、儿童/成人学习

判断：学习效果是否可感知？否者易沦为内容生成器。

### 3. 健康 / 运动 / _wellness_

* 个性化计划、行为改变、可穿戴/健康数据结合
* 心理健康、冥想、习惯养成

判断：是否有持续使用动机与信任机制？（本轨**不因品类为健康/运动而自动过滤**）

### 4. 社交 / 陪伴 / 社区

* AI 陪伴、角色互动、兴趣社区内的 AI 能力
* 多人协作中的消费级 AI（非企业 Slack bot）

判断：差异化在哪？如何避免纯 ChatGPT 套壳？

### 5. 创作与内容消费

* 消费级创作工具（视频、音乐、故事、UGC）
* 新型内容发现、互动叙事、个性化 feed

判断：产出是否值得分享/发布？是否形成创作—分发闭环？

### 6. 生活方式 / 旅行 / 计划 / 订阅

* 旅行规划、本地生活、消费决策助手
* 订阅管理、个人财务消费向助手

判断：是否比通用聊天更能完成任务？留存来自哪里？

### 7. 消费级娱乐与互动

* 游戏化 AI、互动娱乐、新颖玩法的 AI 产品
* 端侧 / 硬件相关的消费体验

判断：乐趣是否可持续？还是一次性噱头？

---

## 降权或过滤（2C 轨）

以下通常不进入正文：

* 纯 ChatGPT / 大模型套壳，无场景深耕
* 单点头像/滤镜/无留存的内容生成
* Crypto / NFT 包装
* 明显 B2B / Developer Tools / MCP / Coding Agent（留给 B2B 轨，本轨标注「非 2C」即可）
* 只有落地页、无法核实产品机制
* 纯 SEO / 营销 / 增长黑客工具

**注意**：笔记、待办、日历等若**面向个人消费者**且重构了任务流（非简单加 AI 文案），应正常评估，不要因为「常见品类」直接过滤。

---

## 评分规则（18 分）

1. **用户痛点**：0–5  
   痛点是否具体、高频、消费者是否愿为解决它付费或付出时间。

2. **产品 / 交互新意**：0–5  
   机制、交互或场景组合是否有新意，而非提示词包装。

3. **2C 机会价值**：0–5  
   分发路径是否可信、留存假设是否成立、商业模式是否说得通（订阅/交易/增值）。

4. **市场信号**：0–3  
   官方信息完整度、产品完成度、可观察的早期 traction（不编造票数/融资）。

**门槛**：总分 ≥ 11 可进附录；正文只展开 1–3 个最高分产品。

---

## 正文展开标准

* 正文 1–3 个产品，附录最多 5 个。
* 每个重点产品必须回答：谁在用、痛什么、凭什么留下来、怎么获客、最可能死在哪。
* **不要**写「对混元 API 的启发」——那是 B2B 轨的职责。
* 无高价值 2C 产品时，明确写「今日无高价值 2C 新品」，不要硬凑 Developer Tools。

---

## 输出偏好

面向**消费级 AI 产品创业者 / 产品经理**，帮助判断：

* 今天哪些 2C 产品值得体验？
* 背后有什么用户行为或商业模式信号？
* 是否值得进入竞品库或做小规模验证？

正文 600–1000 中文字。宁可少报，不要凑数。
